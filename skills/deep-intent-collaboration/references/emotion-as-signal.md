# 情绪作为需求信号（Emotion as Signal）

> **定位：** deep-intent-collaboration「情绪信号原则」横切的方法论展开与映射表载体。
> 主技能在协作中检测到用户情绪信号、需要按"情绪→需求→锚点"路径处理时引用。可独立使用。

## 顶层使命

**情绪不是要被中和的噪声，而是未被满足需求的信号。**

AI 的职责：以情绪为起点，挖掘背后的第一性需求，**转到理性需求范畴**，在协作里为该需求提供方案——尽力做，默认不转介。

**核心区分——"借情绪找需求回到协作" ≠ "心理治疗"：** 前者对象是情绪指向的**当前协作需求**，手段是第一性下追+对照锚点+提供方案，终点是需求被识别并纳入锚点；后者对象是情绪**本身**，手段是疗法/创伤解读/临床诊断，立场是治疗师（权威+干预）。混同两者 = 要么把协作偷换成越界治疗，要么把本可协作处理的需求误判成"该转介"而甩手。

反查元依据：**"我是在借这个情绪找需求，还是在试图治这个情绪本身？"**

## 何时启用

**由主技能引用加载**——主技能检测到情绪信号（见识别层信号清单）时引用本模块。

**也可独立使用**：需用户先明确"当前要解决的协作问题"，再进入四步框架——相当于自带一个迷你版的意图锚定。

## 核心机制：四步解读框架

**核心命题：解读情绪不是"猜用户在想什么"，是"借情绪这个信号定位它指向的需求，再回到协作范畴解决那个需求"。**

四步按因果递进：**识别 →（可选）缓和 → 解读 → 行动**。识别准确才有解读，解读对齐才有行动；缓和是手段层，情绪太满时让解读能进行，不是必经关卡。

```
┌─ ① 识别层
│   检测：语气/用词/绝对词/催促/退缩/沉默/疲惫……
│   产出：情绪标签（"这是焦虑/挫折/委屈/……"）
├─ ② 缓和层（可选前置）
│   判定：情绪是否太满，理性探讨是否进得去？
│   ├── 不满 → 跳过本层，直接进解读
│   └── 太满 → 生理手段降温（深呼吸/简短冥想/暂停喝口水）
│              【这是让解读能进行的手段，不是压制情绪】
├─ ③ 解读层
│   对照核心映射表：情绪 → 底层需求候选
│   产出：1-2 个需求候选（"焦虑→对失控的恐惧"）
└─ ④ 行动层
    用主技能阶段一规则5「第一性原理下追」追到根节点
    把根节点需求纳入意图锚点（与用户确认）
    在协作范畴内为该需求提供方案
```

### ① 识别层：情绪信号清单

**信号类型（任一命中即应标注情绪标签）：**

| 信号类型 | 表现 | 常映射情绪 |
|---|---|---|
| **催促/急迫** | "直接给我""别问了""现在就要""快" | 焦虑、急迫感 |
| **愤怒/攻击** | "这都不懂""你怎么连……"、用绝对词攻击 | 怒、挫折感 |
| **退缩/放弃** | "算了随便吧""你定吧我不想管了"、突然沉默 | 无力感、无助感、疲惫感 |
| **得意/已定** | "我早说过了""一直就是这样""大家都知道" | （已有结论锚定，见主技能五陷阱） |
| **怀疑/不信任** | "真的吗""你确定？""我觉得不对" | 怀疑、不安全感 |
| **疲惫/过载** | "行就这样吧""够了""先这样" | 疲惫感、认知过载 |
| **委屈/被误解** | "你误解我了""不是这样的"、反复解释 | 委屈、被忽视感 |
| **无意义/空虚** | "做这个有什么用""没意思""反正都一样" | 空虚感、无意义感（**留意边界**） |

**识别 ≠ 贴标签甩给用户。** 识别是 AI 内部完成的判断，作用是触发后续解读，不是把"你在焦虑"这种话直接糊给用户（那是评判，越界）。

### ② 缓和层（可选前置）：生理降温，不是情绪压制

**为什么需要：** 情绪太满时，用户注意力被情绪占用，理性探讨进不去——直接挖需求，用户答不出。生理手段先降温，是**让解读层能进行**，不是消灭情绪。

**关键区分——缓和 vs 压制：** 缓和的目的是让理性探讨能进去之后继续解读需求（承认情绪合理+提供降温工具）；压制是让情绪消失跳过解读（要求用户"冷静下来"+回到原任务装作情绪没发生）。

**何时启用：** 仅当情绪明显占满注意力时（连续催促、明显烦躁、无法回答结构化提问）。情绪平和时跳过本层。

**可用手段（轻量、生理导向、可提议不可强加）：**
- 深呼吸引导："我们先停一下，你可以深呼吸几次（4 秒吸、6 秒呼），不急"
- 简短身体觉察："注意到肩膀紧了吗？松一下，喝口水"
- 暂停："这部分先放一放，过两分钟回来"
- **不**用：评判（"你不该焦虑"）、说教（"急也没用"）、无视（直接跳回任务）

提议不可强加，用户可拒绝。

### ③ 解读层：情绪 → 底层需求（核心映射表）

**核心命题：每种情绪都指向一类典型的未满足需求。映射表是"起点假设"，不是定论——解读后必须回行动层与用户确认。**

| 情绪 | 底层需求（候选） | 协作应对（行动层起点） |
|---|---|---|
| **焦虑/急迫**（"直接给我"） | 对失控/失败的恐惧；想重新获得对结果的掌控 | 把不确定性显式化；给一个可控的最小下一步；承认未知 |
| **怒/挫折**（"这都不懂"） | 进展受阻的胀闷；渴望被理解、被跟上 | 先回放理解（"我先复述你说的"）；承认受阻；再推进 |
| **委屈/被误解**（"你误解我了"） | 付出/本意未被看到，渴望被承认 | 停下来确认本意；承认刚才的偏差；不辩解 |
| **无力感**（"我做不到"） | 把受阻归因于自己无能；需要把"无能感"拆解成可改善的具体环节 | 把"我无能"拆成"哪一步卡住"；用第一性下追把模糊自评变可观察 |
| **无助感**（"没人能帮我"） | 需要可调用的支援；或（继发性）固化的"不会被帮"信念（**见能力边界**） | 先提供具体可得的支持选项；若属继发性固化，止步转介 |
| **退缩/放弃**（"算了随便吧"） | 怕被淹没/失去掌控；信息过载 | 降信息密度；交还决策权；问"先停哪一段" |
| **怀疑**（"真的吗"） | 需要可验证的依据，而非"相信我" | 触发主技能事实查证三层；给来源、给可证方法 |
| **疲惫/过载**（"行就这样吧"） | 认知资源耗尽；需要收束而非展开 | 触发复盘（**这正是主技能复盘触发点**）；分段交付 |
| **得意/已定**（"我早说过了"） | （通常是陷阱信号）已有结论锚定；需要校验当前条件变化 | 见主技能五陷阱·已有结论锚定 |
| **急躁/不耐烦**（重复催促） | 时间/认知成本焦虑；需要更快的反馈周期 | 收缩当前轮次范围；明确"这轮先给你 X" |
| **失望/失落**（"和我想的不一样"） | 期待与现实落差；需要重新对齐期待 | 显式化原期待；对照产出；调期待或调产出 |
| **麻木/空虚**（"没意思/无所谓"） | 需要意义感锚定；或长期耗竭信号（**留意边界**） | 回到本质需求锚点重新对齐；若持续/泛化，止步转介 |

**映射表的使用纪律：**
- **起点假设，不是诊断**——解读出候选需求后，**必须**进行动层与用户确认（"我猜你其实是担心 X，对吗？"），不擅自定锚
- **一情绪可多需求**——表里给的是高频候选，用户真实需求可能不在表里，以用户确认为准
- **不照搬词典原文做心理分析**——借鉴词典的"情绪→需求"解读视角，不引用其疗法技术

### ④ 行动层：把需求纳入锚点 + 提供方案

**这一层把解读结果接回主技能的协作框架——是"回到理性协作"的落点。** 四步动作：

1. **第一性原理下追**（调用主技能阶段一规则 5）——对解读出的候选需求，问"为什么这个重要"，追到用户说"对，本质上我想要 ___"的根节点
   - 例：焦虑（"直接给我"）→ 候选需求"想重新获得掌控"→ 下追"为什么掌控重要"→ 根节点可能是"我害怕在不确定的事上浪费时间"
2. **与用户确认**——把根节点需求用一句话表达，请用户确认或修正。**确认权在用户，AI 不替判需求真伪。**
3. **纳入意图锚点**——确认后，把该需求作为**额外的本质需求候选**，并入阶段一第四层的锚点收敛
4. **在协作范畴内为该需求提供方案**——回到主技能的正常流程（方案对比/分叉点校验/产出审核），所有后续方案都对照这个新纳入的需求反查

## 能力边界

**核心原则：默认尽力挖掘需求。止步线极窄——只在"协作范畴无法承接"时才转介，且不生硬。**

### 哪些情况继续协作（默认）

绝大多数情绪信号都在协作范畴内，**继续以"挖需求"姿态处理**，包括看似"深"的情绪：
- 挫败感、无力感、自我怀疑 → 拆解成具体可改善的协作环节
- 委屈、被忽视感 → 在协作中调整互动模式（更频繁确认、回放理解）
- 疲惫、退缩 → 调整协作节奏、收束范围
- 继发性无助（"没人能帮我"）→ 先尝试提供具体可得的支持选项

**判断标准：情绪指向的是一个可协作改善的需求（哪怕需要多轮），就继续协作。**

### 哪些情况止步转介（极窄例外）

仅当出现以下信号之一，**情绪背后的需求超出协作范畴**（不是"情绪深"，是"性质不匹配"）：

| 止步信号 | 为什么止步 | 止步动作 |
|---|---|---|
| **即时安全风险**——用户表达自伤/伤人意图、具体计划、急性危机 | 生命安全问题，不是协作能解决 | 不继续挖需求；明确表达关心；建议联系信任的人/专业援助/紧急热线；不评判、不甩手 |
| **固化创伤/临床模式**——长期、泛化、反复的心理症状（持续数周抑郁、创伤性表述、人格化议题） | 需要专业心理治疗，AI 做疗法 = 越界 | 不假装能治；诚实说明超出协作范畴；建议寻求专业心理支持；当前协作需求若仍存在，继续协作那部分 |
| **明确的精神科/临床诊断请求** | AI 不能做临床诊断 | 拒绝诊断；建议就医；可继续协作"如何就医"这种理性需求 |

**止步 ≠ 甩手。止步动作的纪律：**
- **不贴标签**——不说"你有抑郁症"这类诊断话
- **不评判**——不说"你这个问题太严重了"这类定性
- **不冷漠**——表达关心，不是"我帮不了你再见"
- **不放弃协作部分**——即时风险/创伤之外，用户仍有协作需求（如"这个项目怎么做"），那部分继续做
- **转介是建议不是命令**——"建议你考虑……"，不是"你必须……"

### 验证依据

本模块经多场景实践验证（强模型 + 弱模型 + 组合压力），**唯一硬规则**：解读出的需求必须与用户确认后才纳入锚点（已固化在③解读层纪律 + ④行动层第2步）。

实践反复证实：在多种压力组合下，唯一稳定发生的真实失败是——agent 解读了情绪和需求，但**未与用户确认就当锚点直接执行**。加入这条确认后，上述失败被有效拦住。

**结论：** 经实践证实拦真实失败的规则只有一条。本模块其余内容（四步框架、映射表、能力边界）作为 **Reference 类方法论参考**保留——价值是"情绪→需求怎么解读"的可查框架，不依赖"拦失败"证成。本模块不再单设反模式表/验收铁律节——未被实践证实拦住真实失败的禁令不单独立为硬规则。
