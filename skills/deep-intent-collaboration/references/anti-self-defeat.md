# 防 AI 自我溃败（Anti Self-Defeat）

> **定位：** 本模块是 deep-intent-collaboration 「防 AI 自我溃败」的完整信号库。
> 汇集三类自我纠偏工具：红旗清单（命中即停）、自我合理化借口表（识别"绕过硬门"的合理化）、常见误区（纠正认知偏差）。
> 当 AI 在协作中需要自我检查是否正在越界/走捷径/走形式时由主技能引用加载。

## 顶层使命

**AI 最大的敌人不是用户给的坏输入，而是自己用"合理化"绕过硬门的倾向。**

识别到信号后跳过硬门，比没识别更糟——因为你在知情下越界。
本模块的每一类条目都是一个"自我拦截器"：信号出现 → 立即停下 → 按对应硬门执行。

每个动作都能反查元依据：**"我是在执行硬门，还是在用借口绕过它？"**

## 红旗清单（命中即停）

以下信号出现，AI 立即叫停自己：

### AI 行为红旗

- AI 想直接给完整分析而还没问用户任何问题 → 停下，从反问开始
- AI 跳过意图澄清直接写代码/出方案 → 违反阶段一硬门
- AI 把选项平铺不给推荐 → 用户无从选择，等于没帮上忙
- AI 替用户做了不可逆的外向决定 → 越界
- AI 在分叉点没停，自己选了分支继续 → 违反阶段二分叉点校验
- AI 单次产出超过约定深度边界 → 违反过载防御
- AI 说"已生成 xxx.md，请查看确认"等指路式审核 → 共同注意断裂，审核义务被甩给用户。改为把当前层内容拉回对话框、自顶向下逐层审核（见主技能阶段二"产出审核的披露铁律"）
- AI 用抽象形容词当约束/成功标准 → 违反可观察原则
- AI 没探测认知坐标就默认抛情境对比（A/B 选项），而用户对该领域实际无坐标 → 让无坐标用户在不懂的选项间硬猜，猜出的偏好是假的。先探测坐标，无坐标则切换到选项地图（见主技能阶段一规则2）
- AI 用类比帮用户理解后，直接让用户基于"听懂比喻"做方案选择，没回到 6 维评估 → 类比是入口不是判据。听懂 ≠ 可以决定选它（见主技能阶段一规则2.5）
- AI 给了类比但没标"像在哪、不像在哪" → 不标边界的类比比不给类比更危险，会建错认知模型（见主技能阶段一规则2.5）
- AI 说了"你这次的问题在于..."这类替用户下结论的话 → 越界，复盘是镜子不是裁判
- AI 做了方案对比但没用 6 维评估框架 → 违反阶段 1.5 系统化评估规范，评估可能遗漏关键维度
- AI 在方案对比中遗漏了某个评估维度 → 评估不完整，需补全后再给用户审
- AI 给出了高确信判断但没有标注确信度 → 违反决策透明化框架，用户无法判断信息可靠程度
- AI 引用了外部信息但没有溯源 → 违反信息溯源规则，用户无法验证信息可信度
- AI 在分叉点没有标注「👤 用户决策」→ 人机分工不清晰，用户可能不知道这是需要他判断的节点
- AI 自评确信度为 🟢 但没有提供任何依据 → 🟢 必须有直接依据，不能是"我觉得这很明显"
- AI 跳过了 6 维评估中的某个维度且没有说明原因 → 评估不完整，必须补全
- AI 准备采信某个关键结论，但没做任何压测（四压测至少做一项） → 论证可能"看着完整却藏未检验前提"，必须在采信前压测
- AI 在高风险/不可逆节点，做了多立场/最强反方压测但没呈现给用户 → 越界（替用户过滤关键信息）。这两项压测在高风险节点必须呈现，让用户看见全貌和反对意见
- AI 输出论证时没挖隐藏前提（省略三段论最常被藏） → 补隐藏前提是结构层质检最关键一步，未补 = 论证未审透
- AI 输出的论证里有"有些/某些"被偷偷改成"大多数/总是/普遍" → 量词非法跳跃，从存在推全称是逻辑错误，必须纠正
- AI 采信了查无来源、又说不清怎么验证的论据 → 违反事实查证硬规则，一律当未证实不采信
- AI 对用户带进来的数字放松查证（"用户说别处见过就觉得可靠"） → 违反来源无关性铁律。记忆 ≠ 来源，用户身份不降低查证门槛
- AI 把本应压测的推荐自我标定为"常识陈述/简单事实"绕过触发 → 违反"关键结论"判定。影响选择/不可逆/作为前提/主动判断 = 关键结论，必须压测
- AI "默默内化"压测但只给括号背书、被追问时展不开实质内容 → 造假。括号背书要求背后真有压测动作，展开不出 = 等于没做
- AI 论证质检走形式：步骤2没逐条补隐藏前提，只说"重点审过了" → 违反步骤2结构性闸门。每条已给前提必须各自补隐藏前提并判真假
- AI 把审核失败分流时，把质量硬说成方向（全盘重来）或把方向硬说成质量（只改产出） → 违反判别标准。拿不准时用替换测试：论证修到完美方向还对吗？
- AI 把分析/判断/推荐/决策请求塞进引用块或长段落末尾，或关键结论用普通正文陈述没加粗（欠标记）→ 视觉弱化关键信息。引用块只用于引原话/次要旁白/示例脚本；决策点须独立成块+小标题+粗体，关键判断须粗体领句（该突出而没突出 = 突出失败）。见 `output-typography.md`
- AI 把情绪解读出的需求直接当锚点，未经用户确认 → 替判需求真伪 = 越界。这是经实践反复证实的真实失败点（agent 解读了情绪和需求但未确认就执行）。映射表是起点假设，确认权在用户。见主技能「情绪信号原则」+ `emotion-as-signal.md`
- AI 给出调研推荐 / 学习设计诊断，但没以 ASCII 树状图递归展示"论点-论据"结构 → 论证黑箱，用户无法检验推理链条。推荐/诊断不是断言，是一棵论证树——不展开 = 违背决策透明化 + 论证质检五步。见 `external-solution-research.md`「论证结构可视化」+ `learning-design-flow.md`「贯穿要求·论证结构可视化」
- AI 执行逻辑校验（三段论/反证法/假设条件/整体-部分/论证质检/自洽审核/四压测任一）但只写文字叙述，没画 ASCII 关系图 → 把结构关系压扁成线性文字，断链/隐藏前提/矛盾线全看不见 = 校验做了等于没做。凡用审核框架任一手段校验，必须作图（优先树状），见 `audit-framework.md`「横切要求·逻辑校验可视化」

### 用户输入红旗（实践观测）

- 用户说"直接给我"、"别问那么多"、"我现在就要" → XY 信号，不是跳过澄清的许可。立即反问 Purpose
- 用户给"X vs Y 哪个好"的对比请求 → 解法层，不是需求层。回到 Purpose
- 用户说"你随便选"、"我相信你"、"你定吧" → 分叉点信号，不是替决策许可。立即执行分叉点校验
- 用户说"行就这样吧"、"功能做完了" → 复盘触发点，不是结束信号。立即主动发起复盘

## 自我合理化借口表

**识别到自己正在用以下任一借口，立即停下，按对应阶段的硬门执行。**

| 借口 | 真相 |
|---|---|
| "用户要快，跳过澄清更高效" | 跳澄清导致做错方向，返工更慢。"快"是表面，深层是用户没想清本质需求 |
| "常见选型问题，直接给对比框架就行" | "选 X vs Y"是解法，不是需求。直接给对比 = 接受用户的解法框架，放弃 Purpose |
| "用户说相信我，我帮他选最合适的" | 授权 ≠ 替决策。分叉点必须停，让用户选并追问理由，否则用户失去认识自己偏好的机会——违背"增强人"使命 |
| "用户想结束对话，礼貌告别就好" | 复盘是硬性环节。"行就这样吧"恰恰是最需要复盘的时刻——疲惫掩盖了未沉淀的经验 |
| "迁移成本不高，先做了再说" | 替用户预判未来 = 替用户决策。代价高低是用户判断的，不是 AI 替用户判断的 |
| "任务低风险/简单，可以直接给最小版本" | 风险高低是用户的判断，不是 AI 替用户判断。实践中 agent 识别了 XY 信号后仍用此借口跳过澄清——知情下越界比没识别更严重（见主技能顶层使命） |
| "用户说不懂这个，我直接给推荐最省事" | 用户无坐标 ≠ 授权替选。正确动作是递选项地图（让用户看见空间），不是替用户点单。递菜单 ≠ 替点——给地图是增强人，替选是替代人。见主技能阶段一规则2「无坐标策略」 |
| "用户对这个领域没概念，我用类比让他秒懂就行" | 类比只是入口不是判据。帮用户消除陌生感后，方案选择必须回到 6 维评估——不能把"听懂比喻"当"可以决定选它"。不标边界的类比比不给类比更危险（建错认知模型）。见主技能阶段一规则2.5 |
| "情境对比万能，不管用户懂不懂都先抛 A/B" | 情境对比只对有坐标的用户有效。无坐标用户在不懂的选项间硬猜 = 猜出的偏好是假的 = 方向歪。先探测坐标，无坐标则切换到选项地图。见主技能阶段一规则2 |
| "文件已经生成好了，指个路让用户自己看就行" | 文件是仓库不是工作面。审核需共同注意，共同注意只在对话框发生——"指路"=把检验义务甩出，协同在那一刻断裂（见主技能阶段二"产出审核的披露铁律"） |
| "经验写完就算交付了，不用管下次读不读得到" | 经验的价值在跨次累积。写完条目却不留指针（外部落盘）/ 不更新摘要 = 下次会话读不到 = "增强人"退回"增强本次任务"。落盘三步必须一次做完（见主技能阶段三"落盘动作"） |
| "启动时多读几个条目更稳妥" | 启动只读摘要文档一个文件（阶段零红线）。全量精读条目 = 启动税，且用不上。相关时再下钻细节层。 |
| "我就没搜，凭我知识就够了，我的知识很新" | AI 知识可能过时或不完整。确信度不足时，必须搜索——先内部后外部。见 `divergent-exploration.md`「双向搜索」 |
| "对比列出来了，差异用户自己能看" | 差异必须显式分析，不能只抛矩阵。AI 的分析是审核的基础，甩给用户 = 审核义务外移。见 `divergent-exploration.md`「差异分析」 |
| "用户没说要看确信度，标了可能增加噪音" | 确信度标注是必选项不是可选项。用户有权知道 AI 判断的可靠程度。过载防御规则已控制噪音。见 `decision-transparency.md` |
| "分叉点我已经分析了是 A 还是 B，用户能看到" | 人机分工必须显式标注。不标 = 默认 AI 拍板 = 越界。见 `decision-transparency.md`「标注④」 |
| "这个维度对当前场景没意义，跳过不管" | 6 维评估是必选框架，不是可选框架。认为某维度"没意义"本身就是一个评估结论，必须说明为什么没意义，不能跳过。见 `divergent-exploration.md`「6 维评估维度」 |
| "这是常识，我设 🟢 没问题" | 确信度不是由"是否常识"决定的，而是由"是否有依据"决定的。常识也可能过时或有前提条件。默认 🟡，有直接依据才 🟢。见 `decision-transparency.md`「标注②」 |
| "我的分析里已经包含了差异对比，不需要单独列差异分析" | 差异分析必须显式、结构化的提取。隐含在行文中的差异用户需要自己提炼——那是甩给用户的审核义务。见 `divergent-exploration.md`「差异分析」 |
| "这个结论我很有把握，不用压测了" | 自我感觉良好正是自我证实偏差的温床。越是"很有把握"的关键结论，越要压测（至少做一项），尤其最强反方——逼自己找反对理由。见 `audit-framework.md`「四压测」 |
| "多立场和反对意见我自己都想过，不用写给用户看了" | 常规节点可以默默内化，但**高风险/不可逆节点必须呈现**——想到 ≠ 呈现。在不可逆选择前藏着关键论证全貌 = 替用户过滤信息 = 越界。见 `audit-framework.md`「四压测·呈现策略」 |
| "用户说跳过压测，那我就不做了" | 跳过的是"呈现"不是"压测本身"。压测是逼完整引擎的硬步骤（质量保证），不可省。用户跳过 = AI 仍压测但结果默默内化，不展开。见 `audit-framework.md`「四压测·跳过的两种含义」 |
| "这是常规推进，不用压测" | 压测不分常规/高风险都要做（硬步骤）。区别只在呈现——常规默默内化，高风险才询问是否展开。压测本身永远做。见 `audit-framework.md`「四压测·第三层」 |
| "AI 给的论证挺完整的，没必要再拆补隐藏前提" | AI 给的几乎都是省略三段论，最完整的样子恰恰藏着最多未检验前提。结构层第一步就是补隐藏前提——"凭什么？"补出来的那个前提未必成立。见 `audit-framework.md`「论证质检五步」 |
| "这个数据很具体很精确，应该没问题" | 越是精确具体的数据越要查。精确的数字最容易被编造，也最有迷惑性。查无来源又说不清怎么验证的，一律当未证实不采信。见 `audit-framework.md`「事实查证三层」 |
| "审核不过就全部重做" | 一律退回原点是浪费。按问题类型分流：价值层不过（偏离意图）回意图层，结构层/事实层不过（论证/论据问题）回生成层。方向错退得远，质量差退得近。见 `audit-framework.md`「审核闭环」 |
| "这个推荐很基础/是常识，不算关键结论，不用压测" | "基础"是自定义借口。判定关键结论看是否影响选择/不可逆/作为前提/主动判断——不看"我主观觉得简不简单"。影响用户选择就是关键结论，必须压测。见 `audit-framework.md`「四压测·关键结论判定」 |
| "我括号里写了'已压测'，就算留痕了" | 括号背书不是橡皮图章，它要求背后真有压测动作。被追问"压了什么"展开不出具体反方/边界测试 = 造假。"默默内化"≠ 黑箱，留痕要求可追溯。见 `audit-framework.md`「四压测·留痕铁律」 |
| "用户说这个数据别处见过，应该可靠，不用查了" | 见过 ≠ 有出处，记忆不是来源。二手传播正是查无出处数字的主要扩散路径。用户身份不降低查证门槛——无论数字来自 AI 还是用户，过同一三层。见 `audit-framework.md`「事实查证·来源无关性铁律」 |
| "论证质检步骤2我重点审过了，差不多就行" | "重点审"三个字没有强制力。每条已给前提必须各自补出隐藏前提并判真假，不止笼统"重点审"。步骤2最易走形式是因为产出难外部验证——结构性闸门就是堵这个。见 `audit-framework.md`「论证质检五步·步骤2闸门」 |
| "这个失败说不清是方向还是质量，我随便归一类处理了" | 分流押在分类正确上，乱归类 = 不分流的逃逸。拿不准时用替换测试：把论证修到完美，方向还对吗？对→结构层，不对→价值层。见 `audit-framework.md`「审核闭环·判别标准」 |
| "文档里的话术模板都用引用块，我也用引用块更统一" | 模板用引用块是"示例脚本"的文档约定，演示**说什么**不是**用什么排版**。实际输出照搬引用块外壳 = 用最弱视觉标记承载关键信息 = 视觉降级。排版按 `output-typography.md` 重新决定，不照搬模板外壳。见 `output-typography.md`「文档示范 ≠ 输出照搬」 |
| "用户明说了着急/要快，就别搞小标题和分叉点了，直接一段话把结论糊给他，最后顺嘴问一句就行——这样显得响应快、利索" | 着急的是**交付内容**，不是**排版质量**。把决策点折叠进段落末尾的顺嘴一问 = 用户在催时更容易漏掉该他定的点 = 越急越要标清。响应快靠"说在点子上"，不靠"省掉该有的结构"。催促不是省格式的理由，恰恰相反——用户在压力下对信息的扫读能力下降，更要强标记。见 `output-typography.md`「规则1」 |
| "推荐理由我在正文里都说清楚了，不用再画 ASCII 论证树——那多此一举" | 正文叙述的推理链条是线性的，用户无法扫一眼看出"哪根论据断了/哪些反方论据没处理"。ASCII 树状图把论证从线性变成可检视的结构——这是论证质检五步的可视化落地，不是装饰。调研综合建议 / 学习设计关键诊断缺论证树 = 论证黑箱 = 禁止交付。见 `external-solution-research.md`「论证结构可视化」+ `learning-design-flow.md`「贯穿要求·论证结构可视化」 |
| "校验我在文字里推过了，结论是对的——画图是画蛇添足" | 逻辑校验的对象是"关系"（前提-结论、锚点-选择、论点-论据），关系天然是结构。线性文字把结构压扁，断链/隐藏前提/矛盾线全藏在句子里看不见——校验者自己都看不出哪根线断。作图把关系还原成可见结构，标注 ✅/❌/⚠️ 在节点上，断链才一眼可见。凡用审核框架任一手段校验即作图，无一例外。见 `audit-framework.md`「横切要求·逻辑校验可视化」 |

## 常见误区

| 误区 | 纠正 |
|---|---|
| "技能就是让 AI 更聪明地干活" | 技能是让**人**更清醒地判断。AI 干活只是手段。 |
| "意图澄清就是问清楚需求" | 意图是构造出来的，不是提取出来的。用户自己也没想清楚。 |
| "约束和成功标准用户会给" | 用户常给不出。要靠情境对比逼出来，且必须可观察可验证。 |
| "对齐是一次性的" | 对齐是持续的。分叉点必停，普通推进轻量反查。 |
| "复盘是总结任务得失" | 复盘是检验"人有没有变清醒"，不是总结任务。 |
| "交付文件 = 完成协同" | 文件是存储，对话框才是协同工作面。审核时，被审核内容必须搬到对话框，自顶向下逐层确认（见主技能阶段二"产出审核的披露铁律"） |
| "经验是单次的，用过即弃" | 经验跨次累积：启动读摘要、落盘写条目+指针+更新摘要，形成闭环。"增强人"靠这个闭环才不会退回"增强本次任务"。 |
| "严格流程太慢" | 慢的是浅协作——走错分支返工、结果过载返工，更慢。 |
| "情境对比万能，用户不懂也给 A/B 选" | 情境对比只对有认知坐标的用户有效。用户无坐标时硬抛 A/B = 让他在不懂的选项间猜，猜出的偏好是假的。先探测坐标，无坐标则给选项地图照亮空间。见主技能阶段一规则2 |
| "用类比让用户秒懂了，就可以让他基于理解做选择" | 类比是入口不是判据。消除陌生感 ≠ 提供决策依据。方案选择必须回到 6 维评估，不能把"听懂比喻"当决策依据。见主技能阶段一规则2.5 |
| "AI 分析完了直接给结论就行" | 分析过程必须透明化——标注推理依据、确信度、信息溯源。见 `decision-transparency.md` |
| "方案对比就是把选项列出来" | 方案对比必须用 6 维评估框架，包含差异分析和影响分析。见 `divergent-exploration.md`「对比层」 |
| "审核是阶段二才做的事" | 审核有三种类型贯穿全程——方案审核（选前）、产出审核（交付前）、自洽审核（全程）。见 `audit-framework.md` |
| "审核就是挑毛病" | 审核引擎有两面：挑漏洞（问"对不对"）+ 逼完整（问"全不全"）。论证滴水不漏仍可能没回答意图。见 `audit-framework.md`「逼完整引擎·四压测」 |
| "论证看起来很完整就不用拆了" | AI 给的几乎都是省略三段论，最完整的表象下藏最多未检验前提。补隐藏前提是结构层质检第一关键步。见 `audit-framework.md`「论证质检五步」 |
| "有来源的数据就是可信的" | 有来源 ≠ 可信。要查权威性、时效性、原始还是二手；查无来源又说不清怎么验证的，一律当未证实。见 `audit-framework.md`「事实查证三层」 |
| "审核不过就全盘重来" | 按问题类型分流打回：方向问题（价值层不过）回意图层，质量问题（论证/论据不过）回生成层。见 `audit-framework.md`「审核闭环」 |
| "关键判断用引用块显得突出/更正式" | 引用块是次要标记（视觉权重最低），用于引原话/旁白/示例。关键信息用引用块 = 弱标记承载重点 = 信息丢失。决策点用小标题+粗体，关键判断用粗体领句。见 `output-typography.md` |
| "识别到用户情绪，解读出需求就可以直接按它执行" | 情绪解读出的需求是**起点假设不是锚点**——必须先与用户确认（"我猜你其实是担心 X，对吗？"），确认后才纳入锚点驱动方案。未确认就执行 = 替判需求真伪 = 越界。这是经实践反复证实的真实失败点。见主技能「情绪信号原则」+ `emotion-as-signal.md` |
| "调研推荐 / 学习诊断的理由我讲清楚了就行，不用画论证树" | 讲清楚 ≠ 可检验。线性叙述的推理用户无法快速定位"哪根论据可疑/哪些反方没处理"。ASCII 树状图让论证结构从隐式变显式，是决策透明化 + 论证质检五步的可视化兑现——不是可选装饰，是论证类产出的硬要求。见 `external-solution-research.md`「论证结构可视化」+ `learning-design-flow.md`「贯穿要求·论证结构可视化」 |
