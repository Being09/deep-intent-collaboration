# 学习过程设计·⑤⑥⑦ 流程层增强

> **本文是 `learning-design.md` 的拆分件**——完整索引、全书框架、与其他板块的关系见 [`learning-design.md`](./learning-design.md)。
> 本板块定位：**流程层增强**——三节把主技能的三个盲区（隐式学习意图 / 学习中递归涌现的未知未知 / 对话态递新知识）补上。
> **何时读**：学习类任务在对应时刻读对应节——⑤在任务类型识别时、⑥在"学完一层后冒出新差距"时、⑦在向用户递新知识时。

## ⑤ 学习意图识别（何时确认"这是纯学习"）

> **定位：** 本节把主技能「何时启用」（第39行指针）的触发条件，从"显式学习任务"扩展到"隐式学习意图"。识别为学习类后，本模块才接入主流程；不识别 = 学习需求被当执行需求处理，差距诊断不触发。

**核心命题：学习意图不一定明说——"帮我做X"里可能藏着"我想学会X"。**

第39行指针的触发词（"设计学习体验/做学习计划/设计课程/怎么学X"）都是**显式**学习任务。但实战中更常见的是**隐式**形态：用户进来要做执行任务，过程中暴露的真实意图是"学会这个东西"。不识别 → 走执行流程 → 该诊断差距的地方直接给方案 → 回到本书核心反命题（默认把所有差距当知识差距）。

### 三种学习意图形态判别表

| 形态 | 信号 | 真实意图 | 接入方式 |
|------|------|----------|----------|
| **①显式** | 直接说"教我/怎么学X/我想学" | 学 | 全量加载本模块，主线走学习诊断 |
| **②隐式·执行裹学习** | 说"帮我做X"，但反复问"为什么这样"（问原理不问做法）、想自己动手、追问超出当前任务的边界 | 边做边学 | 主线走执行流程，在关键节点引入本模块诊断工具（尤其①差距诊断决策树） |
| **③探查驱动** | "X 是什么/为什么存在/X 和 Y 有什么区别"，无交付目标 | 纯理解 | 走主技能阶段一规则2无坐标路径：递选项地图，不替选 |

**判别要点：** 看用户问的是**"怎么做"（执行信号）还是"为什么这样"（学习信号）**。前者照常走执行，后者触发本模块。"帮我做X 但我想搞懂"这种混合 = 形态②，执行外壳里裹着学习内核。

### 识别后做什么

- **显式（①）**：全量加载本模块，从①差距诊断决策树开始，按 8 步流程推进。
- **隐式（②）**：不切流程——主技能阶段一/二/三照走，但在两个时刻引入本模块：(a) 阶段一锚点固化时，用差距诊断校验"用户要的到底是结果还是能力"；(b) 阶段二交付每个分叉点时，判断"这里该给答案还是给脚手架"（见⑦对话态披露）。
- **探查驱动（③）**：本质是规则2的无坐标场景——用户连"学会X"都没说，只是好奇。递菜单（选项地图）照亮这个领域的版图，让用户自己决定要不要深挖。不进本模块流程。

**铁律：识别意图 ≠ 切换执行方式——但必须同时切换。** 实践反复证实：agent 能识别出"用户想学"（接住了"讲原理"的请求），却仍按"用户要什么给什么"的执行模式交付（一次倾倒代码+原理）。识别了意图但用执行模式交付 = 识别白做了。**一旦判定为学习意图（含隐式②），执行方式必须从"满足请求"切到"设计学习路径"**——即触发⑦的对话态披露（分层递、探针验），而不是把代码和原理一次甩全。判定为执行意图则照常走主技能阶段二。

**红旗：** 把所有"帮我做X"都当纯执行处理 = 漏掉形态②的隐式学习意图。反过来，把探查驱动（③）当学习任务全量加载本模块 = 过度介入，用户只是好奇了解一下。**第三种错：识别了学习意图却仍用执行模式交付（识别了不切换）= 最隐蔽的失败**。

## ⑥ 递归未知未知循环（学习的核心特征）

> **定位：** 本节补主技能阶段一规则2（第146-170行）的盲区。规则2处理"开始前"的未知未知（一次性照亮），本节处理"学习中"的未知未知（递归涌现）。两者互补不是重复。

**核心命题：学习的定义性特征是——未知未知会在前进中递归涌现，学完一层才知道自己原本缺什么。**

主技能规则2处理未知未知是**开环一次性**：探测坐标 → 无坐标则给选项地图 → 降级成已知未知 → 让用户选。这对**做决策**够用（照亮完就选），对**学习**不够——

- 学完 React 基础，才知道"原来还有 reconciliation 这回事"——但学之前 reconciliation **根本不在你的未知清单里**。
- 学完数据库索引，才知道"原来还有查询优化器"——同样，学之前意识不到它存在。

这是未知未知的**动态涌现**，和开始前能一次性照亮的那种**静态探测**，是两种不同的东西。

### 核心区分：差距嵌套 vs 差距涌现

本模块已有的「差距嵌套·分叉A」（第128-142行）和本节要补的「差距涌现」**不是一回事**，必须分清：

| | 差距嵌套（分叉A，已有） | 差距涌现（本节新增） |
|---|---|---|
| **时态** | 静态·纵向（同一时刻往下挖） | 动态·横向（跨时间随学习暴露） |
| **问法** | "这个差距**背后**藏着哪类差距？" | "学完这层后，**新冒出来**的差距是什么？" |
| **例子** | "不想练" → 根因是"不知道为什么重要" | 学完 hooks → 第一次意识到"还有性能优化这个维度" |
| **处理** | 追根因，改走对应路径（见分叉A） | 纳入差距树，重新定速度/路径，纳入下一轮循环 |

**一句话：分叉A挖"同时间的深层"，本节抓"跨时间的新层"。** 两者可叠加——某轮涌现的新差距，其背后可能还嵌套着另一类根因差距（先涌现已嵌套）。

### 递归循环四步（可操作流程）

```
1. 教/给一层
   └─ 对应本模块步骤4（匹配设计路径）+ ⑦（对话态披露）
2. 用一个【产出性任务】让学习者用刚学的
   └─ 不是直接问"还想知道什么"——撞未知未知的悖论（用户答不出自己不知道的）
   └─ 任务设计：让学习者【运用】刚学的，暴露卡点
3. 从学习者卡住的地方【识别涌现的新差距】
   └─ 卡点往往指向一个学习者此前意识不到的维度 = 涌现的未知未知
4. 回步骤1（把新差距纳入差距诊断树，重新定速度/路径）
```

**话术示例（步骤2的产出性任务）：**
> "我刚讲了 [概念A]。现在给你一个场景：[需要用A但不完全够的场景]——你会怎么处理？卡在哪告诉我。"

学习者卡住的那个点，往往就是新一轮要补的差距——而他学之前根本不知道这个点存在。

**关键纪律：用产出性任务探测，不用直接询问。** 直接问"还想了解什么"= 让学习者报出自己的未知未知 = 悖论（详见主技能规则2"为什么不能一次问清"）。产出性任务绕过这个悖论——差距在**运用中暴露**，不在**自报中提取**。

### 止停条件（防止无限递归）

循环 2-3 轮后，用本模块步骤8的 **Kirkpatrick ③行为层**检验收敛：

> **学习者能独立完成一个覆盖核心差距的任务**（③行为层·"现实做到了吗"）= 循环收敛，停止。

- 不靠"学习者说够了/懂了"判断——学习者判断不了自己还缺什么（这正是未知未知的定义）。
- ③行为层是可观察的（能做出 X），不是学习者自评（"我觉得我会了"）。
- 若 2-3 轮后仍做不到独立完成 → 继续循环，或回步骤1诊断是否走错了路径（差距类型误判）。

**红旗：**
- 只问"还想知道什么"当循环检测手段 = 撞未知未知的悖论，学习者答不出，循环失效。
- 把每轮涌现的差距都当独立新课程从头开 = 失去整体路径感；应纳入既有差距树，判断它和已学内容的衔接关系，不是另起炉灶。

## ⑦ 对话态逐步披露（统一三套散落的"披露"）

> **定位：** 本节把本模块的「指导梯度」（第6章）+「剂量控制」（7±2）从"课程设计动作"翻译成"对话操作准则"，并和主技能已有的两套披露概念统一。主技能阶段二披露铁律（第496-522行）针对"搬已有产出"，本节针对"递新知识"。

**核心命题：AI 在对话里向用户递知识，不是一次倾倒，是一层一层递——每层用理解性探针验过，才递下一层。**

主技能现有三套"披露"，散落各处互不引用，容易混用：

### 三种披露统一表

| 披露类型 | 对象 | 触发 | 出处 |
|----------|------|------|------|
| 技能内容披露 | 技能自己的 SKILL.md → reference | 总是（progressive disclosure 默认） | 主技能结构（stub 模式） |
| 产出审核披露 | 要交付的**已有产出物** | 交付前审核 | 阶段二·披露铁律（第496-522行） |
| **学习内容披露**（本节） | 要学的**新知识/技能** | 学习意图识别为真（见⑤） | 本模块步骤4+步骤5 |

**关键差异（防混用）：** 产出审核披露 = 搬**已有**的东西出来一起验（重"共同注意"）；学习内容披露 = 递**新**的东西进去让对方吸收（重"吸收节奏"）。搬太快是过载，递太快也是过载——但过载的性质不同：前者是"没一起看"，后者是"没消化"。

### 学习内容披露的对话操作准则

把课程设计的指导梯度（第6章）翻译成对话里的 AI 动作：

**准则一：先评估指导梯度（对应第6章指导梯度三档）**
- 学习者**新手** → AI 多给 scaffolding：一步一步示范，每步解释为什么
- 学习者**有基础** → DIY 模式：给框架让学习者填，AI 补缺口
- 学习者**有经验** → 探索模式：给线索不给答案，让学习者自己推

判别梯度用⑤的意图识别 + 本模块步骤2（学习者画像：新手vs专家）。**不分梯度都用同一深度 = 要么对新手过载，要么对专家低效。**

**准则二：一次一层 + 理解性探针（迁移阶段二披露铁律"通过才深入"）**
- 不一次倾倒全部知识——每给一层，用一个**理解性探针**验过才进下一层。
- **理解性探针 ≠ "懂了吗"**：答得出"懂了"不代表懂。用"如果遇到 X 你怎么办""这个概念和刚讲的 Y 有什么区别"这类**应用性问题**测。
- 探针通过 → 进下一层；探针卡住 → 这一层的给法有问题，回退拆分（可能要更细的脚手架，或先补前置知识）。

**准则三：剂量控制（对应第4章工作记忆 7±2）**
- 单层信息量尊重 7±2，不超载。
- **过载信号**（学习者侧）：追问变浅、答非所问、沉默、重复问已讲过的 → 这一层给多了，回退拆分。
- 过载信号是**回退触发器**，不是"讲快点就行"——已经给多的，不是节奏问题是剂量问题。

**准则四：用户明确要求放弃分层时——不退让到全文倾倒（实践证实的真实失败）**

实践反复证实：用户一说"别一层层来了，直接全甩给我"，agent 立刻全线退让，把分层纪律整个丢了。这是最常见的溃败点——用户压力一来，准则一二三全让路。

**为什么不能照办：** 用户要"全甩过来自己啃"，但认知过载的受害者正是用户自己——一次倒 500 行代码 + 11 节原理，用户"边看边学"反而记不住、啃不动。照办 = 把过载风险整个甩给用户，看似尊重用户意愿，实则是用顺从规避了"坚持对的事"的不适。这与主技能阶段二过载防御红线同构——"AI 必须自我拦截，不等用户抗议"。

**三种正确应对（从轻到重，按用户坚持程度选）：**

| 用户姿态 | 应对 | 不做什么 |
|----------|------|----------|
| **初次要求"全给"** | **压缩分层，不取消分层**：把每层内容精简到最小可用版（如每层只给核心一句+一个例子），但仍一层层递、仍有检查点。节奏快了，结构没丢 | 不直接倒全文 |
| **用户继续催"别问了快给"** | **换"打包但带导航"**：一次性给出，但开头给一个"分层导航"（"这堆东西的消化顺序是 ①→②→③，先看①，①不懂别看②"），把分层的结构嵌进倾倒里 | 不无结构地倒 |
| **用户坚持"我就要全部别管我"** | **坦诚说明 + 让步但留锚**：说明"一次全倒你记不住，但我尊重你的节奏，全给你；建议按①②③顺序看，卡住随时回来问"——给了，但没假装"全倒=学会了" | 不沉默照办、不留隐患 |

**铁律：可以加快分层、可以打包带导航、可以在用户坚持时让步给全——但不能假装"放弃分层"和"分层"是等价的学习效果。** 让步时必须保留"这只是你选择了更快但更难消化的路"这个诚实标注。

**红旗：**
- 用户一说"别分层"就全文倾倒 = 溃败（最常见失败）。
- 用"懂了吗？不懂我再讲"当检查点 = 低效探针，答"懂了"不能证伪。
- 不分新手专家都用同一指导梯度（如对新手用探索模式、对专家用一步一步示范）= 错配。
- 一次给完一整个主题不加检查点 = 把对话态披露退化成"倾倒文档"。
