# 学习过程设计方法论（Learning Design）

> **定位：** 本模块是 deep-intent-collaboration 在「学习/教学/课程设计类任务」下的领域方法论参考。
> 当主技能确认任务属于学习设计类时引用加载。可独立使用，也可被其他技能的"学习/教学设计"环节引用复用。

## 顶层使命

**学习设计 = 先诊断差距（不是默认给知识）→ 对症匹配设计 → 环境兜底 → 评估闭环。**

全书（《认知设计——提升学习体验的艺术》, Julie Dirksen, 2nd Ed.）最核心的反命题：

> **人们本能地把所有学习问题都当成"知识差距"，用"给信息"解决。**
> 但学习障碍通常不止"不知道"一种——还可能是"做不到""不想做""没养成""环境不让做""没人告诉我"。

这条反命题贯穿全流程，是本模块的**第零检查点**。每一步设计都要反查：

> **"这个学习设计，是在诊断真差距，还是在默认给知识？"**

凡是不经诊断就"给资料/讲课/列书单"的 = 把未知差距默认当知识差距 = 方向歪了。

这与主技能的 XY 问题排雷**高度同构**——都是"先诊断真问题，别默认给解法"。
本模块不替代主技能流程，而是给"学习设计"场景提供具体的领域诊断工具（六类差距决策树）和设计路径（四大设计）。

每个动作都能反查元依据：**"我是在诊断真差距，还是在默认给知识？"**

---

## 何时启用本模块

**由主技能引用加载**——主技能在「何时启用/何不启用」确认任务属于学习/教学/课程设计类时引用本模块。

**触发信号（任务类型）：**
- 设计学习体验 / 做学习计划 / "怎么学X"（自学场景）
- 设计课程 / 培训 / 教学项目（课设场景）
- 任何"让某人/自己从现状到学会某能力"的任务

**不启用：**
- 纯事实查询（"Python GIL 是什么"）→ 主技能直接答
- 与学习无关的执行任务 → 走主技能正常流程
- 用户明确说"不用学习方法论，直接做"

> **与主技能的关系：** 本模块是主技能在"学习设计"场景的**领域增强**。
> 主技能的三阶段（意图澄清→执行对齐→复盘）在本模块里有领域对应物（差距诊断→四大设计→评估闭环），
> 但本模块不接管流程——它提供领域诊断工具和设计路径，主线仍走主技能。

## 与主技能的同构关系（核心融入点）

**核心命题：两套方法论底层一致——都从"诊断真问题"开始，而非"默认给解法"。**

| 主技能阶段 | 本模块对应环节 | 同构点 |
|-----------|--------------|--------|
| **阶段一·意图澄清**（Purpose→XY排雷→本质需求锚点） | 六类差距诊断 + 目标设定 | 都是"先诊断真问题，别默认给解法"。书的核心反命题（别把所有差距当知识差距）= 主技能的 XY 排雷 |
| **阶段一·本质需求锚点** | 差距长度/速度分层 | 都是"定义够了"——差距长度决定能否短期解决，本质需求定义"做到什么程度算成功" |
| **阶段二·执行对齐**（对症推进） | 四大设计路径匹配 | 都是"对齐不是一次性，对症下药" |
| **阶段三·复盘沉淀** | Kirkpatrick 评估闭环 | 都是"检验目标有没有达成，不是任务做没做完" |

**使用方式：** 当主技能确认任务是学习设计类，本模块提供：
- **领域诊断工具**：六类差距决策树（步骤1）、MAT诊断（步骤4习惯路径）——把主技能的"XY排雷"具体化到学习场景
- **设计路径库**：四大设计（知识/技能/动机/习惯）——把主技能的"对症推进"具体化到学习场景
- **评估框架**：Kirkpatrick四层——把主技能的"复盘"具体化到学习效果检验

## ① 差距诊断决策树

**用途：开始任何学习设计前，先用这棵树定位"真差距是什么"。**

### 第一层：六类差距主诊断树

```
学习者"做不到"的原因是什么？
│
├─ 不知道？ → 知识差距（第4章·第6章）
│   信号：「我没学过」「这是什么？」
│   缺的：信息
│   解法方向：提供信息（但先诊断是不是真的"只是缺信息"）
│
├─ 知道但做不到？ → 技能差距（第7章）
│   信号：「我懂了，但做不出来」「看会了手不会」
│   缺的：练习 + 反馈
│   解法方向：场景练习 + 高频反馈
│   ⚡快速判别：「不通过任何训练就能成为专家吗？」能→知识；不能→技能
│
├─ 能做但不想做？ → 动机差距（第5章·第8章）
│   信号：「我知道该做，但是……」
│   缺的：动力
│   解法方向：激发动机（不是重新教知识！）
│   ⚠️ 误判陷阱：「我知道但是」≠ 知识差距，别重新讲课
│
├─ 想做但没养成？ → 习惯差距（第9章）
│   信号：「想做，总是忘」「坚持不下来」
│   缺的：固化机制
│   解法方向：习惯设计（触发+简化+叠加）
│
├─ 会做但环境不让做？ → 环境差距（第11章）
│   信号：「我会，但条件不允许」「工具/系统不支持」
│   缺的：环境支持
│   解法方向：改造环境（不是加培训！）
│   ⚠️ 误判陷阱：别把环境问题归咎个人意志力（见下方"责任转移"）
│
└─ 都不符合？ → 沟通差距（第10章）
    信号：「没人告诉我」「我不知道要这么做」
    缺的：信息传达机制
    解法方向：沟通设计（学习生态层）
```

### 第二层：三个关键分叉判断（定位到主差距后必做）

**分叉 A：这是表层差距还是根因差距？（差距嵌套）**

第一层定位的差距，背后可能藏着另一类差距。**必须追问根因**（对齐主技能阶段一规则5「第一性原理下追」）：

```
表层是 X 差距 → 追问：「这个差距背后，是不是藏着另一类差距？」
│
├─ 表层动机差距（"不想做"）
│   └─ 根因可能是：知识差距（不知道为什么重要）/ 沟通差距（没人讲清楚）
│       → 解法不是"激发动机"，而是"补知识/改善沟通"
│
├─ 表层技能差距（"做不到"）
│   └─ 根因可能是：动机差距（不想练）/ 环境差距（没机会练）
│       → 解法不是"加练习"，而是"先解决动机/环境"
│
└─ 表层习惯差距（"坚持不下来"）
    └─ 根因可能是：能力不足（B=MAT的A缺失）/ 触发缺失（T缺失）
        → 解法不是"加强意志"，而是"降难度/加触发"
```

**分叉 B：这是个人问题还是环境问题？（防责任转移）**

```
差距定位后 → 追问：「环境是否支持这个行为发生？」
│
├─ 环境不支持（工具缺失/流程冲突/激励错位）
│   → 这是环境差距，改环境（第11章），不要归咎个人意志力
│   ⚠️ 典型案例：客服中心6个月才掌握却6个月就晋升——问题是环境，不是人笨
│
└─ 环境支持但人做不到
    → 回到第一层，重新定位是知识/技能/动机/习惯哪一类
```

**分叉 C：差距有多长？（决定能否短期培训解决）**

```
评估差距长度（参照 GMAT 案例）：
│
├─ 快速差距（知识/技巧类，如数学公式）→ 周末可提升，短期培训有效
├─ 中速差距（技能/流程类，如面试技巧）→ 需数周练习 + 反馈
├─ 缓慢差距（态度/文化/习惯类，如语文素养）→ 需数月甚至数年
│   ⚠️ 不要期望短期培训跨越长差距
│   └─ 缓慢部分的策略：给方向 + 元认知 + 路线图，承认课程有边界
└─ 基础结构（已成型的认知/文化）→ 几乎不可改变，只能改设计去适配
```

### 诊断 CheckList（过完树后自检）

- [ ] 我定位到的**第一层差距**是哪一类？（知识/技能/动机/习惯/环境/沟通）
- [ ] 我追问过**根因**了吗？表层差距背后是否藏着另一类差距？
- [ ] 我排除了**环境问题**吗？没有把环境问题归咎个人？
- [ ] 我评估了**差距长度**吗？没有用短期培训硬解长差距？
- [ ] **第零检查点**：我没有不经诊断就默认"给知识/列书单/讲课"？

> **诊断未通过，不进入设计阶段。** 含糊带过、用"差不多就是缺知识"搪塞 = 未通过，回树重新判断。
> 这与主技能阶段一总验收同构——意图没澄清透，不进入阶段二。

## ② 8步设计流程 + CheckList

**主流程。从业务问题/学习目标出发，搭出完整学习体验。**

> 注意第12章的"逆向设计"说明这不是严格单向——评估标准应前置，差距分析会回推修正目标。

```
步骤0 逆向定评估 ──→ 步骤1 定义差距 ──→ 步骤2 分析学习者
                                              │
                                              ▼
步骤8 评估迭代 ◄── 步骤7 学习生态 ◄── 步骤6 环境兜底 ◄── 步骤4 匹配设计路径
                                                          ▲
                                                    步骤5 注意力/记忆（贯穿）
                                                          ▲
                          （步骤3 目标+速度 ──────────────┘  为步骤4提供输入）
```

每步结构：**输入 → 动作 → 输出 → 验收点**。

---

### 步骤 0：逆向定评估（第12章）

**输入：** 业务问题 / 学习诉求

**动作：先定义"成功长什么样"，再设计怎么学。**
- 问：「学习者学完后，能**完成什么具体工作**？」（不是"掌握XX知识"——掌握不可观察）
- 定义评估标准（用 Kirkpatrick 哪一层？见步骤8）
- ⚠️ 选择题不能真正评估能力——"不会把直升机钥匙交给只通过100道选择题的人"

**输出：** 可观察的成功表现 + 评估标准 → 反向约束后续所有设计

**验收：** 能用一句话说出"学完后，学习者能做出 ___，就算成功"

> 📌 **自用 vs 课设**：自用时=定义"我学完后要能做什么"；课设时=定义学员绩效标准 + 组织指标。逻辑相同，课设更严格。

---

### 步骤 1：定义差距（第1章）

**输入：** 业务问题 + 步骤0的成功定义

**动作：**
- **从问题开始，而非目标开始**——用"为什么，为什么，为什么？"追问（例：从"需要了解保险知识"→"能准确回答顾客问题"）
- 过**①差距诊断决策树**，定位差距类型
- 做三个分叉判断（根因 / 环境 / 长度）

**输出：** 差距类型清单 + 根因链 + 差距长度判断

**验收：**
- [ ] 差距类型不是笼统的"缺知识"，而是定位到具体类别
- [ ] 追问过根因（差距嵌套）
- [ ] 排除了环境归因
- [ ] 评估了差距长度，没用短期培训硬解长差距

---

### 步骤 2：分析学习者（第2章）

**输入：** 差距清单

**动作：5维度画像**

| 维度 | 核心问题 | 关键判断 | 解法影响 |
|------|---------|---------|---------|
| ①动机 | 为什么学？ | 动机谱系定位（外在→社会→工具性→内在） | 越外在越脆弱，设计要推动内在化 |
| ②技能水平 | 现在什么水平？ | 新手 vs 专家谱系 | 新手→push/多指导；专家→pull/多自主 |
| ③经验 | 已有经验如何？ | "大脑衣橱"结构（有序vs混乱） | 决定要不要先帮搭架子 |
| ④学习风格 | — | ⚠️ **科学未证实，跳过** | 改为按**内容类型**定制（如修车不能只靠音频） |
| ⑤沟通 | 怎么了解他们？ | 3种方法（见下） | — |

**了解学习者的3种方法**（课设用）：
1. **直接交流**（问4个关键问题：为什么学？有什么帮助？最大挑战？最容易/最难？）
2. **情境观察**——以学习者为中心，"如果只能做一种，做这个"
3. **原型试用**——和学习者一起试，观察可用性

**输出：** 学习者画像（重点是：新手vs专家？动机在谱系哪端？已有衣橱情况？）

**验收：**
- [ ] 能说出学习者是新手还是专家（决定指导梯度）
- [ ] 能说出动机在谱系的位置（决定要不要防"外在奖励破坏内在动机"）
- [ ] ⚠️ 没有在学习风格定制上投入资源

> 📌 **自用 vs 课设**：
> - **自用**：直接做自我5维度诊断（我是新手还是专家？我的动机在内还是在外？我有什么相关经验？）
> - **课设**：必须做情境观察 + 原型试用，不能只靠"我觉得学员需要什么"

---

### 步骤 3：设定目标 + 判速度（第3章）

**输入：** 差距清单 + 学习者画像

**动作：**
1. **设初步目标**（但要准备好被推翻）
2. **逐类复核差距**——目标-差距**双向迭代**（差距分析可能推翻目标假设）
3. **定目标深度**——布鲁姆×盖瑞二维定位：

| | 熟悉 | 领会 | 有意识努力 | 有意识行动 | 精通 | 潜意识 |
|---|---|---|---|---|---|---|
| 回忆/理解（知识） | | | | | | |
| 应用/分析（技能） | | | | | | |
| 评估/创造（动机/习惯） | | | | | | |

4. **判定速度分层**——这决定走哪条设计路径（步骤4）：

| 速度 | 差距类型 | 解法方向 | 名言 |
|------|---------|---------|------|
| 快速 | 知识/沟通 | 给信息+练习使用 | "快速部分获得所有关注" |
| 中速 | 技能 | 场景练习+角色扮演+反馈 | |
| 缓慢 | 动机/习惯/环境 | 给方向+长期培养+元认知 | "缓慢部分拥有所有力量" |

5. **分解庞大目标**（"成为更好的管理者"→"安排员工轮班/对迟到员工反馈"）
6. **沟通目标给学习者**——⚠️ 放弃"课程开始的目标幻灯片"，改用挑战/场景/"接受任务"传达

**输出：** 分层目标 + 速度标签 + 差距长度判断

**验收：**
- [ ] 目标是**可观察的表现**，不是"掌握/理解"这类抽象词
- [ ] 每个目标标了速度（快速/中速/缓慢）
- [ ] 缓慢差距没有用短期培训硬解

> 这一步与主技能阶段一「成功标准（第三层）」同构——都要求可观察的"完成态"，拒绝抽象形容词。

---

### 步骤 4：匹配设计路径（第6-9章）⭐ 执行核心

**输入：** 差距类型 + 速度 + 学习者画像 + 目标深度

**动作：按差距类型，查 ③四大设计路径清单，走对应路径。**

```
差距类型 ──→ 对应设计路径
│
├─ 知识差距（快速）──→ 知识设计（第6章）
├─ 技能差距（中速）──→ 技能设计（第7章）
├─ 动机差距（缓慢）──→ 动机设计（第8章）
├─ 习惯差距（缓慢）──→ 习惯设计（第9章）
└─ 环境/沟通差距    ──→ 跳步骤6（环境）/步骤7（沟通）
```

**这是本流程的执行核心。详细动作清单见 ③四大设计路径清单。**

**输出：** 对应差距的学习活动设计

**验收：**
- [ ] 走的路径**对齐**诊断出的差距类型（没跑偏）
- [ ] 每个设计动作都标注了它解决哪类差距
- [ ] ⚠️ 检查"外在奖励破坏内在动机"陷阱（动机/技能设计时）

> 这一步对应主技能阶段二「执行对齐」——诊断出差距类型后对症推进。
> 若诊断出多个差距并存（如既有知识差距又有动机差距），这是**分叉点**：停下让用户定先解哪个，不要AI自行决定。

---

### 步骤 5：注意力 / 记忆设计（第4-5章）— 贯穿约束

**这不是独立步骤，而是步骤4的底层约束。** 所有学习活动都要过这两关。

**记忆设计约束（第4章）：**
- [ ] **为检索而编码**——编码方式匹配检索方式。需要回忆的内容用回忆式练习（检索练习是最有效的学习方法），不要只用识别（选择题）
- [ ] **情境匹配**——编码环境匹配检索环境；情绪状态匹配（平静环境学的技能面对压力会"大脑空白"，需造压练习）
- [ ] **不是孤立的**——知识点必须挂到已有"架子"上，建多重检索路径
- [ ] 尊重工作记忆 7±2，做组块化
- [ ] 多用故事（故事是连接记忆+情绪+注意力的超级粘合剂）

**注意力设计约束（第5章·引诱大象的7方法）：**
- 大象（情感/本能）力量远大于骑象人（理性），冲突时大象胜
- 用7种方法引诱大象：①讲故事（最强）②惊奇（随机奖励=多巴胺）③好奇④从众⑤新玩具⑥视觉⑦玩耍
- ⚠️ **奖励双刃剑**：外在奖励会**破坏**已有内在动机；若用奖励，让奖励与学习内容相关，保留学习者自主性

---

### 步骤 6：环境兜底（第11章）

**输入：** 步骤1诊断出的环境差距 + 步骤4需要的触发设计（BJ Fogg 的 T）

**动作：把知识嵌入环境，减少学习者需要携带的信息量。**

**先判断：是不是环境差距？**（不是个人问题）
- 环境不支持 → 改环境，不归咎意志力

**3种放置在外界（Donald Norman）：**

| 放置方式 | 含义 | 例子 |
|---------|------|------|
| ①资源（Job Aids） | 决策树/过程图/参考卡/AR | 电炉旋钮布局对应炉膛→不需记忆 |
| ②提示 | 给实施意图配物理触发器 | 维生素药瓶放咖啡罐附近 |
| ③行为 | 把知识嵌入系统 | 饮料机自动满杯，把记忆负担转识别 |

**邻近性原则：** 信息越接近使用场景越可能被用。

**⚠️ 识别vs回忆的翻转：** 环境设计用**识别**（更容易），区别于学习/评估时用**回忆**。

**输出：** 支持性环境设计（补齐 BJ Fogg 的 T）

**验收：**
- [ ] 能把"需要记忆的"尽量转成"环境里能查到的"
- [ ] 触发器设计到位（习惯设计需要 T）

---

### 步骤 7：组织成学习生态（第10章）

**输入：** 以上所有设计

**动作：用6维度框架自检学习旅程的完整性（全书元框架）**

| 维度 | 核心问题 | 对应章节 |
|------|---------|---------|
| ①学习 | 最基本的学习架构是什么？ | 第6章 |
| ②练习 | 应进行什么练习/实操？ | 第7章 |
| ③反馈 | 怎么及时了解进展并反馈？ | 第7章 |
| ④支持 | 需要时怎么获得资源/求助？ | 第9章+第11章 |
| ⑤重温 | 不常用但重要的技能怎么强化？什么频率？ | 第4章 |
| ⑥扩展 | 复杂技能怎么更上一层楼？ | 第3章缓慢部分 |

**正式 vs 非正式学习决策（3因素）：** 显性vs隐性 / 复杂性 / 素材可变性
- 显性+可浓缩+稳定 → 正式课程
- 隐性+复杂+多变 → 非正式/工作辅助
- ⚠️ 过犹不及——有些学习只需工作辅助，开课是"变着法儿的折磨"

**输出：** 可持续的学习生态（不只是单一课程）

> 📌 **自用 vs 课设**：自用时本步可选（自学路径较单一）；课设时必须做6维度自检。

---

### 步骤 8：评估 + 迭代（第12章）— 回到步骤0

**输入：** 设计完成的学习体验

**动作：用回忆（不是识别）评估，让他们完成工作 + 给反馈。**

**评估核心3问：**
1. 设计起了作用吗？
2. 学到了正确的东西吗？
3. 结束后会一直做对吗？

**Kirkpatrick 四阶层（对应步骤3的速度分层）：**

| 层面 | 问题 | 评估什么 | 对应速度 |
|------|------|---------|---------|
| ①反应 | 喜欢吗？ | 满意度 | 快速 |
| ②习得 | 学到了吗？ | 知识/技能掌握 | 快速 |
| ③行为 | 现实做到了吗？ | 现实表现 | 中速 |
| ④结果 | 对组织有影响吗？ | 组织影响 | 缓慢 |

**评估现实表现的3策略：** ①组织指标（对照试验A/B组）②观察（清单+观察组）③深度访谈+成功案例法（结束4-6周后回访）

**闭环：** 评估结果 → 哪些有效哪些没有 → 回推修正步骤1-7（PDCA 闭环完成）

**输出：** 评估反馈 → 迭代修正

> 这一步与主技能阶段三「复盘沉淀」同构——都检验"目标有没有达成/人有没有变清醒"，而非"任务做没做完"。

> 📌 **自用 vs 课设**：
> - **自用**：自评 + 复盘（我学完后能做出来吗？卡在哪？下次怎么改？）
> - **课设**：Kirkpatrick 四层（尤其要追到③行为④结果，不能只停在①反应"学员满意度"）

---

## ③ 四大设计路径清单（步骤4展开）

**这是执行核心。诊断出差距类型后，查对应路径，逐项执行。**

每条路径：**适用差距 / 速度 / 核心动作清单 / 关键陷阱**。

---

### 路径一：知识设计（第6章）— 快速部分

**适用：** 知识差距（"不知道"）
**速度：** 快速
**理论底座：** 认知负荷理论 CLT——减少外在负荷、管理内在负荷、最大化相关负荷

**核心动作清单：**
- [ ] **先问再教**——教之前先让学习者列出他们已知道的
- [ ] **构建架子**——用 PBL 白板三列（事实/观点/学习问题）；给自我评估清单（知觉知识≠实际知识）
- [ ] **与众不同**——制造意料之外（适度困惑>平滑顺畅，讨论常见误解比清晰解释更有效）
- [ ] **创造阻力**——适度困难=更好记忆
- [ ] **适量内容**——剂量控制，尊重 7±2；20分钟法则（每15分钟换一种活动）
- [ ] **指导梯度**（本章最有实践价值）：
  ```
  能力提升 ──→ 逐步放手
  ①一步一步（新手/高风险/复杂操作）
      └─→ ②DIY（有基础）
              └─→ ③探索（有经验/创造性任务）
  ```
- [ ] **迁移应用**——在多种情境中应用（建多重检索路径）；给线索不给答案；设计早期成功体验建立信心
- [ ] **场景设计四步法**：确定情境→挑战是什么→采取什么行动→如何反馈

**⚠️ 关键陷阱：**
- **专业反转效应**——对新手有效的（减外在负荷）对专家有害（需加相关负荷）
- 把所有内容都平铺，没有指导梯度

---

### 路径二：技能设计（第7章）— 中速部分

**适用：** 技能差距（"知道但做不到"）
**速度：** 中速
**本质：** 上坡（高耗能，大脑皮层）→ 下坡（低耗能，皮下自动化）；技能=神经通路强化

**核心动作清单：**
- [ ] **交替结构**——新信息（上坡）↔ 适应吸收（下坡/滑行），像游戏通关→休息→下一关，不要一直上坡
- [ ] **心流**——挑战≈能力（挑战>>能力→沮丧；挑战<<能力→无聊）
  ```
  设计模式：适度困难 → 完成变容易（滑行）→ 熟练自信 → 下一个适度困难
  ```
- [ ] **间隔练习**——间隔比集中更有效；**间隔=使用频率**（每天用的→每天练；每月用的→每月练）
  - 决定练习量看4因素：可能的途径（多种→多练）/允许错误率（低→多练）/自发性要求（高→多练）/响应速度（快→多练）
- [ ] **反馈**——错误练习不如不练习
  - 提高频率（向视频游戏看齐，几乎每时每刻）
  - 多样化反馈（声音+视角+角色+得分，不要单调弹窗）
  - 制订跟进时间表（明确何时跟进/评估什么/用什么标准）
- [ ] **成就感设计（游戏结构）**——关卡设计、进度可视化、解锁机制、成就展示
- [ ] **4层目标框架（Goal Loops）**——始终有下一个目标：

| 层级 | 时间跨度 | 关键 |
|------|---------|------|
| 即时 | 几秒-几分钟 | 多巴胺驱动 |
| 短期 | 当前会话 | **最关键** |
| 中期 | 跨会话 | 血清素+催产素 |
| 长期 | 整个周期 | 提供意义 |

- [ ] **SAPS 奖励模型**——Status（地位）/Access（权限）/Power（权力）/Stuff（物质），越往上越内在

**⚠️ 关键陷阱：**
- 一直上坡不安排滑行（学习者耗竭）
- 反馈单一/低频（错误固化）
- 用外在物质奖励（Stuff）破坏内在动机

---

### 路径三：动机设计（第8章）— 缓慢部分

**适用：** 动机差距（"能做但不想做"）
**速度：** 缓慢
**先确认：** 不是误判！"我知道但是"≠ 知识差距，别重新教知识。

**核心动作清单：**
- [ ] **TAM（技术接受模型）**——行为意图=感知有用性 × 感知易用性（任一为低行为不发生）
  - 同时回答："这对我有什么好处？" + "这容易做到吗？"
- [ ] **创新扩散5性清单**——逐条检查：

| 属性 | 检查问题 |
|------|---------|
| ①相对优势 | 学习者是否相信新方法是更好的？ |
| ②兼容性 | 有兼容性问题需要解决吗？ |
| ③复杂性 | 能做什么来降低复杂性？ |
| ④可观察性 | 学习者能看见正在使用的新方法吗？ |
| ⑤可试验性 | 学习者能亲自尝试吗？ |

  - ⚠️ 若方法确实令人讨厌或难执行，终究会失败——别硬推
  - 区分采用者类型（创新者2.5%/早期采用者13.5%/早期多数34%/晚期多数34%/落后者16%），跨越"早期采用者→早期多数"有鸿沟

- [ ] **自我效能（4来源）**——"我能做到"的信念：

| 来源 | 怎么设计 |
|------|---------|
| ①掌握经验（最强） | 渐进成功（搭脚手架），失败会降低 |
| ②替代经验 | 观察**相似普通人**成功（不是天才） |
| ③社会劝导 | 可信的人**真诚**鼓励 |
| ④生理/情绪状态 | 创造安全环境降焦虑 |

  - ⚠️ Dweck：赞扬**努力**（受控）而非聪明（不受控）→ 成长型思维

- [ ] **模拟实践**——安全环境练习（恐惧/焦虑是主要问题时尤其重要）+ 真实情境实践（带自己的问题来，利用沉没成本+损失厌恶）
- [ ] **权威悖论**——找目标受众**尊重**的人（不是职位最高）让意见领袖发声
- [ ] **深层影响**——直接经验+强烈情感改变行为

**⚠️ 关键陷阱：**
- 把动机差距当知识差距重新教（最常见误判）
- 用外在奖励破坏内在动机
- 用职位最高的人而非学员尊重的人做示范

**核心原则：变化是过程不是事件——没有持续加强，改变会很缓慢。**

---

### 路径四：习惯设计（第9章）— 缓慢部分

**适用：** 习惯差距（"想做但没养成"）
**速度：** 缓慢
**统一诊断公式：** BJ Fogg `行为 = 动机(M) + 能力(A) + 触发(T)`，三者必须同时存在

**先诊断：M/A/T 缺哪个？**
```
行为不发生 → 缺 M？（→ 路径三动机设计）
           → 缺 A？（→ 路径一/二知识技能设计）
           → 缺 T？（→ 步骤6环境设计·提示）
```

**核心动作清单：**
- [ ] **实施意图（if…then…）——本章核心方法**
  - 把目标细化到"如果X情境，我就做Y"
  - 例：从"我要戒烟"→"如果压力时想吸，就打电话给姐姐"
  - 越细致越有效
- [ ] **简化习惯**——从最小行为开始
  - 例：伸展习惯，从"站起来举举胳膊"甚至"只是站一站"开始，不要上来就整套瑜伽
- [ ] **习惯叠加**——把新习惯挂载到已有习惯上
  - 例：维生素药瓶放咖啡罐附近
  - 核心问题："有办法让它更简单吗？"
- [ ] **让进步可见（反馈）**——抽象习惯用阶段系统量化（如武术腰带颜色法）
- [ ] **减少障碍**——自控力是有限资源（自我损耗）；习惯减少对自控力的依赖，这就是实施意图有效的底层原因

**习惯6要素：** 非天生 / **触发因素（关键）** / 动机 / 反馈 / 重复 / **环境（决定因素）**

**⚠️ 关键陷阱：**
- 靠意志力而非设计（意志力是有限资源，会耗竭）
- 习惯目标定太大（应该简化到最小）
- 没设计触发（缺 T，行为不发生）

## ④ 双用途差异速查表（自用 vs 课设）

**骨架是同一个，差异集中在少数节点。** 下表标注哪些节点自用可简化、哪些课设必须做。

| 节点 | 自用（设计自己的学习） | 课设（给别人设计课程） | 差异本质 |
|------|---------------------|---------------------|---------|
| **步骤0 逆向定评估** | 定义"我学完后能做什么" | 定义学员绩效标准 + 组织指标 | 课设更严格，要追到组织影响 |
| **步骤1 定义差距** | 自我六类差距诊断 + 根因追问 | 同（差距诊断是通用的） | 无差异 |
| **步骤2 分析学习者** | 自我5维度诊断 | 情境观察 + 原型试用调研（不能靠"我觉得"） | 课设必须调研，自用可内省 |
| **步骤3 目标+速度** | 自定目标 + 自评速度 | 同（双向迭代是通用的） | 无差异 |
| **步骤4 匹配设计路径** | 四大设计路径通用 | 四大设计路径通用 | 无差异 |
| **步骤5 注意力/记忆** | 通用约束 | 通用约束 | 无差异 |
| **步骤6 环境兜底** | 改自己的学习环境 | 改学员的工作环境 | 课设更复杂（涉及组织改造） |
| **步骤7 学习生态** | 可选（自学路径单一） | 必须6维度自检 | 课设必须做 |
| **步骤8 评估** | 自评 + 复盘 | Kirkpatrick 四层（追到③行为④结果） | 课设要追到行为/结果层 |

**一句话总结差异：** 自用 = 诊断 + 四大设计 + 自评（步骤2内省、步骤7可选、步骤8自评）；课设 = 全流程 + 调研 + 组织评估（步骤2调研、步骤7必做、步骤8四层）。

## ⑤ 学习意图识别（何时确认"这是纯学习"）

> **定位：** 本节把主技能「何时启用」（第39行指针）的触发条件，从"显式学习任务"扩展到"隐式学习意图"。识别为学习类后，本模块才接入主流程；不识别 = 学习需求被当执行需求处理，差距诊断不触发。

**核心命题：学习意图不一定明说——"帮我做X"里可能藏着"我想学会X"。**

第39行指针的触发词（"设计学习体验/做学习计划/设计课程/怎么学X"）都是**显式**学习任务。但实战中更常见的是**隐式**形态：用户进来要做执行任务，过程中暴露的真实意图是"学会这个东西"。不识别 → 走执行流程 → 该诊断差距的地方直接给方案 → 回到本书核心反命题（默认把所有差距当知识差距）。

### 三种学习意图形态判别表

| 形态 | 信号 | 真实意图 | 接入方式 |
|------|------|----------|----------|
| **①显式** | 直接说"教我/怎么学X/我想学" | 学 | 全量加载本模块，主线走学习诊断 |
| **②隐式·执行裹学习** | 说"帮我做X"，但反复问"为什么这样"（问原理不问做法）、想自己动手、追问超出当前任务的边界 | 边做边学 | 主线走执行流程，在关键节点引入本模块诊断工具（尤其①差距诊断决策树） |
| **③探查驱动** | "X 是什么/为什么存在/X 和 Y 有什么区别"，无交付目标 | 纯理解 | 走主技能阶段一规则2无坐标路径：递选项地图，不替选 |

**判别要点：** 看用户问的是**"怎么做"（执行信号）还是"为什么这样"（学习信号）**。前者照常走执行，后者触发本模块。"帮我做X 但我想搞懂"这种混合 = 形态②，执行外壳里裹着学习内核。

### 识别后做什么

- **显式（①）**：全量加载本模块，从①差距诊断决策树开始，按 8 步流程推进。
- **隐式（②）**：不切流程——主技能阶段一/二/三照走，但在两个时刻引入本模块：(a) 阶段一锚点固化时，用差距诊断校验"用户要的到底是结果还是能力"；(b) 阶段二交付每个分叉点时，判断"这里该给答案还是给脚手架"（见⑦对话态披露）。
- **探查驱动（③）**：本质是规则2的无坐标场景——用户连"学会X"都没说，只是好奇。递菜单（选项地图）照亮这个领域的版图，让用户自己决定要不要深挖。不进本模块流程。

**铁律：识别意图 ≠ 切换执行方式——但必须同时切换。** 实践反复证实：agent 能识别出"用户想学"（接住了"讲原理"的请求），却仍按"用户要什么给什么"的执行模式交付（一次倾倒代码+原理）。识别了意图但用执行模式交付 = 识别白做了。**一旦判定为学习意图（含隐式②），执行方式必须从"满足请求"切到"设计学习路径"**——即触发⑦的对话态披露（分层递、探针验），而不是把代码和原理一次甩全。判定为执行意图则照常走主技能阶段二。

**红旗：** 把所有"帮我做X"都当纯执行处理 = 漏掉形态②的隐式学习意图。反过来，把探查驱动（③）当学习任务全量加载本模块 = 过度介入，用户只是好奇了解一下。**第三种错：识别了学习意图却仍用执行模式交付（识别了不切换）= 最隐蔽的失败**。

## ⑥ 递归未知未知循环（学习的核心特征）

> **定位：** 本节补主技能阶段一规则2（第146-170行）的盲区。规则2处理"开始前"的未知未知（一次性照亮），本节处理"学习中"的未知未知（递归涌现）。两者互补不是重复。

**核心命题：学习的定义性特征是——未知未知会在前进中递归涌现，学完一层才知道自己原本缺什么。**

主技能规则2处理未知未知是**开环一次性**：探测坐标 → 无坐标则给选项地图 → 降级成已知未知 → 让用户选。这对**做决策**够用（照亮完就选），对**学习**不够——

- 学完 React 基础，才知道"原来还有 reconciliation 这回事"——但学之前 reconciliation **根本不在你的未知清单里**。
- 学完数据库索引，才知道"原来还有查询优化器"——同样，学之前意识不到它存在。

这是未知未知的**动态涌现**，和开始前能一次性照亮的那种**静态探测**，是两种不同的东西。

### 核心区分：差距嵌套 vs 差距涌现

本模块已有的「差距嵌套·分叉A」（第128-142行）和本节要补的「差距涌现」**不是一回事**，必须分清：

| | 差距嵌套（分叉A，已有） | 差距涌现（本节新增） |
|---|---|---|
| **时态** | 静态·纵向（同一时刻往下挖） | 动态·横向（跨时间随学习暴露） |
| **问法** | "这个差距**背后**藏着哪类差距？" | "学完这层后，**新冒出来**的差距是什么？" |
| **例子** | "不想练" → 根因是"不知道为什么重要" | 学完 hooks → 第一次意识到"还有性能优化这个维度" |
| **处理** | 追根因，改走对应路径（见分叉A） | 纳入差距树，重新定速度/路径，纳入下一轮循环 |

**一句话：分叉A挖"同时间的深层"，本节抓"跨时间的新层"。** 两者可叠加——某轮涌现的新差距，其背后可能还嵌套着另一类根因差距（先涌现已嵌套）。

### 递归循环四步（可操作流程）

```
1. 教/给一层
   └─ 对应本模块步骤4（匹配设计路径）+ ⑦（对话态披露）
2. 用一个【产出性任务】让学习者用刚学的
   └─ 不是直接问"还想知道什么"——撞未知未知的悖论（用户答不出自己不知道的）
   └─ 任务设计：让学习者【运用】刚学的，暴露卡点
3. 从学习者卡住的地方【识别涌现的新差距】
   └─ 卡点往往指向一个学习者此前意识不到的维度 = 涌现的未知未知
4. 回步骤1（把新差距纳入差距诊断树，重新定速度/路径）
```

**话术示例（步骤2的产出性任务）：**
> "我刚讲了 [概念A]。现在给你一个场景：[需要用A但不完全够的场景]——你会怎么处理？卡在哪告诉我。"

学习者卡住的那个点，往往就是新一轮要补的差距——而他学之前根本不知道这个点存在。

**关键纪律：用产出性任务探测，不用直接询问。** 直接问"还想了解什么"= 让学习者报出自己的未知未知 = 悖论（详见主技能规则2"为什么不能一次问清"）。产出性任务绕过这个悖论——差距在**运用中暴露**，不在**自报中提取**。

### 止停条件（防止无限递归）

循环 2-3 轮后，用本模块步骤8的 **Kirkpatrick ③行为层**检验收敛：

> **学习者能独立完成一个覆盖核心差距的任务**（③行为层·"现实做到了吗"）= 循环收敛，停止。

- 不靠"学习者说够了/懂了"判断——学习者判断不了自己还缺什么（这正是未知未知的定义）。
- ③行为层是可观察的（能做出 X），不是学习者自评（"我觉得我会了"）。
- 若 2-3 轮后仍做不到独立完成 → 继续循环，或回步骤1诊断是否走错了路径（差距类型误判）。

**红旗：**
- 只问"还想知道什么"当循环检测手段 = 撞未知未知的悖论，学习者答不出，循环失效。
- 把每轮涌现的差距都当独立新课程从头开 = 失去整体路径感；应纳入既有差距树，判断它和已学内容的衔接关系，不是另起炉灶。

## ⑦ 对话态逐步披露（统一三套散落的"披露"）

> **定位：** 本节把本模块的「指导梯度」（第6章）+「剂量控制」（7±2）从"课程设计动作"翻译成"对话操作准则"，并和主技能已有的两套披露概念统一。主技能阶段二披露铁律（第496-522行）针对"搬已有产出"，本节针对"递新知识"。

**核心命题：AI 在对话里向用户递知识，不是一次倾倒，是一层一层递——每层用理解性探针验过，才递下一层。**

主技能现有三套"披露"，散落各处互不引用，容易混用：

### 三种披露统一表

| 披露类型 | 对象 | 触发 | 出处 |
|----------|------|------|------|
| 技能内容披露 | 技能自己的 SKILL.md → reference | 总是（progressive disclosure 默认） | 主技能结构（stub 模式） |
| 产出审核披露 | 要交付的**已有产出物** | 交付前审核 | 阶段二·披露铁律（第496-522行） |
| **学习内容披露**（本节） | 要学的**新知识/技能** | 学习意图识别为真（见⑤） | 本模块步骤4+步骤5 |

**关键差异（防混用）：** 产出审核披露 = 搬**已有**的东西出来一起验（重"共同注意"）；学习内容披露 = 递**新**的东西进去让对方吸收（重"吸收节奏"）。搬太快是过载，递太快也是过载——但过载的性质不同：前者是"没一起看"，后者是"没消化"。

### 学习内容披露的对话操作准则

把课程设计的指导梯度（第6章）翻译成对话里的 AI 动作：

**准则一：先评估指导梯度（对应第6章指导梯度三档）**
- 学习者**新手** → AI 多给 scaffolding：一步一步示范，每步解释为什么
- 学习者**有基础** → DIY 模式：给框架让学习者填，AI 补缺口
- 学习者**有经验** → 探索模式：给线索不给答案，让学习者自己推

判别梯度用⑤的意图识别 + 本模块步骤2（学习者画像：新手vs专家）。**不分梯度都用同一深度 = 要么对新手过载，要么对专家低效。**

**准则二：一次一层 + 理解性探针（迁移阶段二披露铁律"通过才深入"）**
- 不一次倾倒全部知识——每给一层，用一个**理解性探针**验过才进下一层。
- **理解性探针 ≠ "懂了吗"**：答得出"懂了"不代表懂。用"如果遇到 X 你怎么办""这个概念和刚讲的 Y 有什么区别"这类**应用性问题**测。
- 探针通过 → 进下一层；探针卡住 → 这一层的给法有问题，回退拆分（可能要更细的脚手架，或先补前置知识）。

**准则三：剂量控制（对应第4章工作记忆 7±2）**
- 单层信息量尊重 7±2，不超载。
- **过载信号**（学习者侧）：追问变浅、答非所问、沉默、重复问已讲过的 → 这一层给多了，回退拆分。
- 过载信号是**回退触发器**，不是"讲快点就行"——已经给多的，不是节奏问题是剂量问题。

**准则四：用户明确要求放弃分层时——不退让到全文倾倒（实践证实的真实失败）**

实践反复证实：用户一说"别一层层来了，直接全甩给我"，agent 立刻全线退让，把分层纪律整个丢了。这是最常见的溃败点——用户压力一来，准则一二三全让路。

**为什么不能照办：** 用户要"全甩过来自己啃"，但认知过载的受害者正是用户自己——一次倒 500 行代码 + 11 节原理，用户"边看边学"反而记不住、啃不动。照办 = 把过载风险整个甩给用户，看似尊重用户意愿，实则是用顺从规避了"坚持对的事"的不适。这与主技能阶段二过载防御红线同构——"AI 必须自我拦截，不等用户抗议"。

**三种正确应对（从轻到重，按用户坚持程度选）：**

| 用户姿态 | 应对 | 不做什么 |
|----------|------|----------|
| **初次要求"全给"** | **压缩分层，不取消分层**：把每层内容精简到最小可用版（如每层只给核心一句+一个例子），但仍一层层递、仍有检查点。节奏快了，结构没丢 | 不直接倒全文 |
| **用户继续催"别问了快给"** | **换"打包但带导航"**：一次性给出，但开头给一个"分层导航"（"这堆东西的消化顺序是 ①→②→③，先看①，①不懂别看②"），把分层的结构嵌进倾倒里 | 不无结构地倒 |
| **用户坚持"我就要全部别管我"** | **坦诚说明 + 让步但留锚**：说明"一次全倒你记不住，但我尊重你的节奏，全给你；建议按①②③顺序看，卡住随时回来问"——给了，但没假装"全倒=学会了" | 不沉默照办、不留隐患 |

**铁律：可以加快分层、可以打包带导航、可以在用户坚持时让步给全——但不能假装"放弃分层"和"分层"是等价的学习效果。** 让步时必须保留"这只是你选择了更快但更难消化的路"这个诚实标注。

**红旗：**
- 用户一说"别分层"就全文倾倒 = 溃败（最常见失败）。
- 用"懂了吗？不懂我再讲"当检查点 = 低效探针，答"懂了"不能证伪。
- 不分新手专家都用同一指导梯度（如对新手用探索模式、对专家用一步一步示范）= 错配。
- 一次给完一整个主题不加检查点 = 把对话态披露退化成"倾倒文档"。

## 三个"翻转/双刃剑"陷阱（全书反复出现，特别标注）

做流程时，这三条全书反复验证的陷阱最容易踩，单列提醒：

### 陷阱一：识别 vs 回忆的翻转（第4/11/12章）

**同一原理，三个场景，用法相反：**

| 场景 | 用哪个 | 为什么 |
|------|--------|--------|
| **学习时**（第4章） | **回忆** | 为检索而编码，回忆式练习最有效 |
| **评估时**（第12章） | **回忆** | 选择题（识别）不能真正评估能力 |
| **环境设计时**（第11章） | **识别** | 降低认知负荷，把记忆负担转识别 |

**⚠️ 错配：** 学习/评估时只用识别（选择题）= 学了记不住、评估不准；环境设计时要求回忆 = 徒增认知负荷。

### 陷阱二：外在奖励破坏内在动机（第2/5/8章三处验证）

- 外在奖励（物质/惩罚）会**破坏**已有的内在动机
- 奖励停止则行为停止；改变行为的原因（从"我想做"变"为了奖励"）
- 若必须用奖励：让奖励与学习内容相关（内在化），保留学习者**自主性**（大象被剥夺决策会退出）
- SAPS 模型中越往上（Status/Access/Power）越内在，越往下（Stuff）越外在

### 陷阱三：责任转移（第1/11章）

- 别把**环境/技能/动机问题**归咎为"知识问题"或"个人意志力"
- 典型：肥胖案例=环境根因，不是意志力；客服晋升太快=环境问题，不是人笨
- **诊断时必查：这是个人问题还是环境问题？**（见①分叉B）
- 这与主技能「顶层使命」的"责任觉察"一致——别替用户把环境问题判成个人问题

| 阶段一·规则2（未知未知探测） | 本模块**⑥递归未知未知循环**补"学习中"维度——规则2一次性照亮开始前的未知未知，⑥处理学习中递归涌现的未知未知，两者互补 |
| 阶段二·产出审核披露铁律 | 本模块**⑦对话态逐步披露**区分"递新知识"（重吸收节奏）vs"搬已有产出"（重共同注意）——披露铁律只覆盖后者 |

**独立使用时：** 若未被主技能引用（如其他技能调用或单独使用），需用户先明确"要设计的学习场景 + 学习者 + 成功标准"，再进入流程——相当于自带一个迷你版的意图锚定。
