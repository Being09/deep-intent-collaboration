# 学习过程设计方法论（Learning Design）

> **定位：** deep-intent-collaboration 在「学习/教学/课程设计类任务」下的领域方法论参考。
> 主技能确认任务属于学习设计类时引用；可独立使用。
> 提炼自《认知设计——提升学习体验的艺术》(Julie Dirksen, 2nd Ed., 机械工业出版社)，每条要素标章节可追溯。

## 顶层使命与核心反命题

**学习设计 = 先诊断差距（不是默认给知识）→ 对症匹配设计 → 环境兜底 → 评估闭环。**

全书最核心的反命题——

> **人们本能地把所有学习问题都当成"知识差距"，用"给信息"解决。**
> 但学习障碍通常不止"不知道"——还可能是"做不到""不想做""没养成""环境不让做""没人告诉我"。

这条反命题贯穿全流程，是**第零检查点**：每步设计反查"是在诊断真差距，还是在默认给知识？"凡是不经诊断就"给资料/讲课/列书单"= 把未知差距默认当知识差距 = 方向歪了。

与主技能的 XY 问题排雷**高度同构**（都是"先诊断真问题，别默认给解法"）。本模块不替代主技能流程，而是给"学习设计"场景提供领域诊断工具和设计路径。

## 何时启用

**触发信号（任务类型）：**
- 设计学习体验 / 做学习计划 / "怎么学X"（自学）
- 设计课程 / 培训 / 教学项目（课设）
- 任何"让某人/自己从现状到学会某能力"的任务

**隐式学习意图也触发**（见下文 ⑤）：用户说"帮我做X"但反复问"为什么这样"（问原理不问做法）= 边做边学。

**不启用：** 纯事实查询（"Python GIL 是什么"）→ 主技能直接答；用户明确说"不用学习方法论，直接做"。

## 与主技能的同构关系

两套方法论底层一致——都从"诊断真问题"开始：

| 主技能阶段 | 本模块对应 | 同构点 |
|-----------|-----------|--------|
| 阶段一·意图澄清（XY排雷） | 六类差距诊断 + 目标设定 | 都"先诊断真问题" |
| 阶段一·本质需求锚点 | 差距长度/速度分层 | 都"定义够了" |
| 阶段二·执行对齐 | 四大设计路径匹配 | 都"对症推进" |
| 阶段三·复盘沉淀 | Kirkpatrick 评估闭环 | 都检验"目标达成"而非"任务做完" |

使用方式：本模块提供领域诊断工具（六类差距决策树）、设计路径库（四大设计）、评估框架（Kirkpatrick）——把主技能的通用原则具体化到学习场景。**主线仍走主技能**，本模块是领域增强。

## ① 差距诊断决策树（开始任何学习设计前先过这棵树）

**用途：定位"真差距是什么"。** 对应主技能阶段一意图澄清（XY 排雷的领域版）。

### 第一层：六类差距主诊断

```
学习者"做不到"的原因是什么？
│
├─ 不知道？ → 知识差距（第4/6章）
│   信号：「我没学过」「这是什么？」  缺的：信息
│
├─ 知道但做不到？ → 技能差距（第7章）
│   信号：「懂了但做不出来」「看会了手不会」  缺的：练习+反馈
│   ⚡ 快速判别：「不通过任何训练能成专家吗？」能→知识；不能→技能
│
├─ 能做但不想做？ → 动机差距（第5/8章）
│   信号：「我知道该做，但是……」  缺的：动力
│   ⚠️ 误判：「我知道但是」≠ 知识差距，别重新讲课
│
├─ 想做但没养成？ → 习惯差距（第9章）
│   信号：「想做，总是忘」「坚持不下来」  缺的：固化机制
│
├─ 会做但环境不让做？ → 环境差距（第11章）
│   信号：「我会，但条件不允许」  缺的：环境支持
│   ⚠️ 误判：别把环境问题归咎个人意志力（见"责任转移"陷阱）
│
└─ 都不符合？ → 沟通差距（第10章）
    信号：「没人告诉我」  缺的：信息传达机制
```

### 第二层：三个关键分叉（定位主差距后必做）

**分叉 A：表层差距还是根因差距？（差距嵌套，对齐主技能规则5 第一性下追）**

```
表层是 X 差距 → 追问：「背后藏着另一类差距吗？」
├─ 表层动机差距（"不想做"）→ 根因可能是知识差距（不知为何重要）/沟通差距
│   → 解法不是"激发动机"，而是"补知识/改善沟通"
├─ 表层技能差距（"做不到"）→ 根因可能是动机差距（不想练）/环境差距（没机会练）
│   → 解法不是"加练习"，而是"先解决动机/环境"
└─ 表层习惯差距（"坚持不下来"）→ 根因可能是 A 缺失/T 缺失
    → 解法不是"加强意志"，而是"降难度/加触发"
```

**分叉 B：个人问题还是环境问题？（防责任转移）**

环境不支持（工具缺失/流程冲突/激励错位）→ 改环境，不归咎意志力。典型案例：客服中心 6 个月才掌握却 6 个月就晋升——问题是环境，不是人笨。

**分叉 C：差距有多长？（决定能否短期培训解决）**

| 长度 | 类型 | 解法 | 备注 |
|------|------|------|------|
| 快速 | 知识/技巧（数学公式） | 周末可提升，短期培训有效 | "快速部分获得所有关注" |
| 中速 | 技能/流程（面试技巧） | 需数周练习 + 反馈 | |
| 缓慢 | 态度/文化/习惯（语文素养） | 给方向 + 元认知 + 路线图 | "缓慢部分拥有所有力量"，别用短期培训硬解 |
| 基础结构 | 已成型的认知/文化 | 几乎不可改，只能改设计去适配 | |

### 诊断 CheckList

- [ ] 第一层差距定位到具体类别（不是笼统"缺知识"）
- [ ] 追问过根因（差距嵌套）
- [ ] 排除了环境归因
- [ ] 评估了差距长度，没用短期培训硬解长差距
- [ ] **第零检查点**：没不经诊断就默认"给知识/列书单/讲课"

**诊断未通过不进入设计阶段。** 含糊带过、用"差不多就是缺知识"搪塞 = 未通过，回树重新判断。

## ② 8 步设计流程（主流程）

**从业务问题/学习目标出发，搭出完整学习体验。** 对应主技能阶段二执行对齐。

```
8 步设计流程（从评估倒推，非严格单向）

  步骤0 逆向定评估
    ↓
  步骤1 定义差距
    ↓
  步骤2 分析学习者
    ↓
  步骤4 匹配设计路径  ← 步骤3（目标+速度）为其提供输入
    ↓
  步骤6 环境兜底
    ↓
  步骤7 学习生态
    ↓
  步骤8 评估迭代

  步骤5 注意力/记忆 — 贯穿全程，不占独立步骤位
```

每步结构：**输入 → 动作 → 输出 → 验收点**。第12章"逆向设计"说明这不是严格单向——评估标准应前置，差距分析会回推修正目标。

**🔗 贯穿要求·论证结构可视化：** 步骤中每个**含论证的产出**（步骤1 差距诊断结论、步骤4 设计路径选择、步骤8 评估判断），必须以结构化信息输出（默认 ASCII 树状图）递归展示"论点-论据"结构。共性规则（形式谱系 / 先图后文 / 三共性要求）见 `output-typography.md`「结构化信息输出」；本场景独有：叶节点标出处（章节 / 学习者画像 / 诊断信号）。

**示例（步骤1 差距诊断的论证树）：**
```
诊断：学习者有[知识差距]（论点）
├─ 论据1：信号匹配——学习者说「我没学过」（第1章知识差距信号）
├─ 论据2：排除其他差距类型
│   ├─ 环境：工具齐全 → 排除环境差距
│   ├─ 动机：主动想学 → 排除动机差距
│   └─ 快速判别「不训练能成专家吗」→ 能 → 确认是知识非技能
└─ 反论据：表层似动机差距（"不想做"）→ 追问根因是"不知为什么重要" → 实为知识差距
```

### 步骤 0：逆向定评估（第12章）

**动作：先定义"成功长什么样"，再设计怎么学。**
- 问：「学完后，能**完成什么具体工作**？」（不是"掌握XX知识"——掌握不可观察）
- ⚠️ 选择题不能真正评估能力——"不会把直升机钥匙交给只通过100道选择题的人"

**验收：** 能用一句话说出"学完后能做出 ___，就算成功"。

### 步骤 1：定义差距（第1章）

**动作：** 从问题开始而非目标开始（"为什么，为什么，为什么？"追问）；过①差距诊断决策树；做三个分叉判断（根因/环境/长度）。

**验收：** 差距类型定位到具体类别；追问过根因；排除环境归因；评估了差距长度。

### 步骤 2：分析学习者（第2章）

**5 维度画像：**

| 维度 | 关键判断 | 解法影响 |
|------|---------|---------|
| ①动机 | 动机谱系（外在→社会→工具性→内在） | 越外在越脆弱，设计推动内在化 |
| ②技能水平 | 新手 vs 专家 | 新手→push/多指导；专家→pull/多自主 |
| ③经验 | "大脑衣橱"结构（有序vs混乱） | 决定要不要先搭架子 |
| ④学习风格 | ⚠️ **科学未证实，跳过** | 改为按**内容类型**定制 |
| ⑤沟通 | 课设用：直接交流/情境观察/原型试用 | — |

**自用 vs 课设：** 自用做自我诊断；课设必须做情境观察 + 原型试用，不能靠"我觉得"。

### 步骤 3：设定目标 + 判速度（第3章）

**动作：** 设初步目标（准备好被推翻）→ 逐类复核差距（目标-差距**双向迭代**）→ 定目标深度（布鲁姆×盖瑞二维）→ **判定速度分层**（决定步骤4走哪条路径）。

| 速度 | 差距类型 | 解法方向 |
|------|---------|---------|
| 快速 | 知识/沟通 | 给信息 + 练习使用 |
| 中速 | 技能 | 场景练习 + 角色扮演 + 反馈 |
| 缓慢 | 动机/习惯/环境 | 给方向 + 长期培养 + 元认知 |

**验收：** 目标是**可观察的表现**（不是"掌握/理解"这类抽象词）；每个目标标了速度；缓慢差距没用短期培训硬解。

> 与主技能阶段一「成功标准（第三层）」同构——都要求可观察的"完成态"，拒绝抽象形容词。

### 步骤 4：匹配设计路径（第6-9章）⭐ 执行核心

**动作：按差距类型查下文 ③四大设计路径，走对应路径。**

```
知识差距（快速）─→ 知识设计（第6章）
技能差距（中速）─→ 技能设计（第7章）
动机差距（缓慢）─→ 动机设计（第8章）
习惯差距（缓慢）─→ 习惯设计（第9章）
环境/沟通差距    ─→ 跳步骤6（环境）/步骤7（沟通）
```

**验收：** 路径对齐诊断出的差距类型；每个设计动作标注解决哪类差距；检查"外在奖励破坏内在动机"陷阱（动机/技能设计时）。

> 对应主技能阶段二「执行对齐」。若诊断出多差距并存（如既有知识又有动机差距），这是**分叉点**：停下让用户定先解哪个，不 AI 自行决定。

### 步骤 5：注意力 / 记忆设计（第4-5章）— 贯穿约束

**不是独立步骤，而是步骤4的底层约束**，所有学习活动都要过这两关。

**记忆设计约束（第4章）：**
- [ ] **为检索而编码**——编码方式匹配检索方式，需要回忆的用回忆式练习（检索练习是最有效的学习方法），不要只用识别（选择题）
- [ ] **情境匹配**——编码环境匹配检索环境；情绪状态匹配（平静环境学的技能面对压力会"大脑空白"，需造压练习）
- [ ] 知识点挂到已有"架子"上，建多重检索路径
- [ ] 尊重工作记忆 7±2，做组块化
- [ ] 多用故事（故事是连接记忆+情绪+注意力的超级粘合剂）

**注意力设计约束（第5章·引诱大象7方法）：**
大象（情感/本能）力量远大于骑象人（理性），冲突时大象胜。7 种引诱方法：①讲故事（最强）②惊奇（随机奖励=多巴胺）③好奇④从众⑤新玩具⑥视觉⑦玩耍。
⚠️ **奖励双刃剑**：外在奖励**破坏**已有内在动机；若用奖励，让奖励与学习内容相关，保留学习者自主性。

### 步骤 6：环境兜底（第11章）

**先判断：是不是环境差距？**（不是个人问题）→ 环境不支持就改环境，不归咎意志力。

**3 种放置在外界（Donald Norman）：**

| 放置方式 | 含义 | 例子 |
|---------|------|------|
| ①资源（Job Aids） | 决策树/过程图/参考卡/AR | 电炉旋钮布局对应炉膛→不需记忆 |
| ②提示 | 给实施意图配物理触发器 | 维生素药瓶放咖啡罐附近 |
| ③行为 | 把知识嵌入系统 | 饮料机自动满杯，把记忆负担转识别 |

**邻近性原则：** 信息越接近使用场景越可能被用。
**⚠️ 识别vs回忆的翻转：** 环境设计用**识别**（更容易），区别于学习/评估时用**回忆**。

### 步骤 7：组织成学习生态（第10章）

**用 6 维度框架自检学习旅程完整性：**

| 维度 | 核心问题 | 章节 |
|------|---------|------|
| ①学习 | 最基本的学习架构？ | 第6章 |
| ②练习 | 应进行什么练习/实操？ | 第7章 |
| ③反馈 | 怎么及时了解进展并反馈？ | 第7章 |
| ④支持 | 需要时怎么获得资源/求助？ | 第9章+第11章 |
| ⑤重温 | 不常用但重要的技能怎么强化？什么频率？ | 第4章 |
| ⑥扩展 | 复杂技能怎么更上一层楼？ | 第3章缓慢部分 |

**正式 vs 非正式学习决策（3因素）：** 显性 vs 隐性 / 复杂性 / 素材可变性。显性+可浓缩+稳定→正式课程；隐性+复杂+多变→非正式/工作辅助。⚠️ 有些学习只需工作辅助，开课是"变着法儿的折磨"。

> 自用时本步可选（自学路径较单一）；课设时必须做 6 维度自检。

### 步骤 8：评估 + 迭代（第12章）— 回到步骤0

**用回忆（不是识别）评估，让他们完成工作 + 给反馈。**

**Kirkpatrick 四阶层（对应步骤3的速度分层）：**

| 层面 | 问题 | 评估什么 | 对应速度 |
|------|------|---------|---------|
| ①反应 | 喜欢吗？ | 满意度 | 快速 |
| ②习得 | 学到了吗？ | 知识/技能掌握 | 快速 |
| ③行为 | 现实做到了吗？ | 现实表现 | 中速 |
| ④结果 | 对组织有影响吗？ | 组织影响 | 缓慢 |

**评估现实表现的 3 策略：** ①组织指标（对照试验 A/B 组）②观察（清单+观察组）③深度访谈+成功案例法（结束 4-6 周后回访）。

**闭环：** 评估结果 → 哪些有效哪些没 → 回推修正步骤 1-7（PDCA 闭环完成）。

> 与主技能阶段三「复盘沉淀」同构——都检验"目标达成/人变清醒"，而非"任务做完"。
> 自用 = 自评 + 复盘；课设 = Kirkpatrick 四层（尤其追到③行为④结果，不能只停在①反应"满意度"）。

## ③ 四大设计路径清单（步骤4展开）⭐ 执行核心

**诊断出差距类型后，查对应路径逐项执行。** 每条路径：**适用差距 / 速度 / 核心动作清单 / 关键陷阱**。

### 路径一：知识设计（第6章）— 快速部分

**适用：** 知识差距（"不知道"）。**理论底座：** 认知负荷理论 CLT（减少外在负荷、管理内在负荷、最大化相关负荷）。

**核心动作清单：**
- [ ] **先问再教**——教之前先让学习者列出他们已知道的
- [ ] **构建架子**——PBL 白板三列（事实/观点/学习问题）；给自我评估清单（知觉知识≠实际知识）
- [ ] **与众不同**——制造意料之外（适度困惑>平滑顺畅，讨论常见误解比清晰解释更有效）
- [ ] **创造阻力**——适度困难 = 更好记忆
- [ ] **适量内容**——剂量控制 7±2；20 分钟法则（每 15 分钟换一种活动）
- [ ] **指导梯度**（本章最有实践价值）：新手/高风险/复杂操作→一步一步；有基础→DIY；有经验/创造性任务→探索
- [ ] **迁移应用**——在多种情境中应用（建多重检索路径）；给线索不给答案；设计早期成功体验建立信心
- [ ] **场景设计四步法**：确定情境→挑战是什么→采取什么行动→如何反馈

**⚠️ 关键陷阱：** 专业反转效应（对新手有效的减外在负荷，对专家有害需加相关负荷）；把所有内容平铺没有指导梯度。

### 路径二：技能设计（第7章）— 中速部分

**适用：** 技能差距（"知道但做不到"）。**本质：** 上坡（高耗能，大脑皮层）→ 下坡（低耗能，皮下自动化）；技能=神经通路强化。

**核心动作清单：**
- [ ] **交替结构**——新信息（上坡）↔ 适应吸收（下坡/滑行），像游戏通关→休息→下一关，不要一直上坡
- [ ] **心流**——挑战≈能力（挑战>>能力→沮丧；挑战<<能力→无聊）。模式：适度困难→完成变容易→熟练自信→下一个适度困难
- [ ] **间隔练习**——间隔比集中更有效；**间隔=使用频率**（每天用的每天练，每月用的每月练）。练习量看 4 因素：可能的途径 / 允许错误率 / 自发性要求 / 响应速度（高→多练）
- [ ] **反馈**——错误练习不如不练习。提高频率（向视频游戏看齐，几乎每时每刻）；多样化反馈（声音+视角+角色+得分，不要单调弹窗）；制订跟进时间表
- [ ] **成就感设计（游戏结构）**——关卡设计、进度可视化、解锁机制、成就展示
- [ ] **4 层目标框架（Goal Loops）**——始终有下一个目标：即时（几秒-几分钟，多巴胺）/ 短期（当前会话，**最关键**）/ 中期（跨会话，血清素+催产素）/ 长期（整个周期，提供意义）
- [ ] **SAPS 奖励模型**——Status（地位）/ Access（权限）/ Power（权力）/ Stuff（物质），越往上越内在

**⚠️ 关键陷阱：** 一直上坡不安排滑行（耗竭）；反馈单一/低频（错误固化）；用外在物质奖励（Stuff）破坏内在动机。

### 路径三：动机设计（第8章）— 缓慢部分

**适用：** 动机差距（"能做但不想做"）。**先确认：** 不是误判！"我知道但是"≠ 知识差距，别重新教知识。

**核心动作清单：**
- [ ] **TAM（技术接受模型）**——行为意图=感知有用性 × 感知易用性（任一为低行为不发生）。同时回答："这对我有什么好处？" + "这容易做到吗？"
- [ ] **创新扩散 5 性清单**——①相对优势（学习者相信新方法更好吗？）②兼容性③复杂性（能降低吗？）④可观察性（能看见使用吗？）⑤可试验性（能亲自尝试吗？）。⚠️ 若方法确实令人讨厌或难执行终究会失败——别硬推。区分采用者类型（创新者2.5%/早期采用者13.5%/早期多数34%/晚期多数34%/落后者16%），跨越"早期采用者→早期多数"有鸿沟
- [ ] **自我效能（4来源）**——"我能做到"的信念：①掌握经验（最强，渐进成功）②替代经验（观察**相似普通人**成功）③社会劝导（可信的人真诚鼓励）④生理/情绪状态（创造安全环境降焦虑）。⚠️ Dweck：赞扬**努力**（受控）而非聪明（不受控）→ 成长型思维
- [ ] **模拟实践**——安全环境练习（恐惧/焦虑是主要问题时尤其重要）+ 真实情境实践（带自己的问题来，利用沉没成本+损失厌恶）
- [ ] **权威悖论**——找目标受众**尊重**的人（不是职位最高）让意见领袖发声
- [ ] **深层影响**——直接经验 + 强烈情感改变行为

**⚠️ 关键陷阱：** 把动机差距当知识差距重新教（最常见误判）；用外在奖励破坏内在动机；用职位最高的人而非学员尊重的人做示范。

**核心原则：变化是过程不是事件——没有持续加强，改变会很缓慢。**

### 路径四：习惯设计（第9章）— 缓慢部分

**适用：** 习惯差距（"想做但没养成"）。**统一诊断公式：** BJ Fogg `行为 = 动机(M) + 能力(A) + 触发(T)`，三者必须同时存在。

**先诊断 M/A/T 缺哪个：** 缺 M→路径三；缺 A→路径一/二；缺 T→步骤6环境设计·提示。

**核心动作清单：**
- [ ] **实施意图（if…then…）——本章核心方法**。把目标细化到"如果 X 情境，我就做 Y"（例："我要戒烟"→"如果压力时想吸，就打电话给姐姐"）。越细致越有效
- [ ] **简化习惯**——从最小行为开始（例：伸展习惯从"站起来举举胳膊"甚至"只是站一站"开始，不要上来就整套瑜伽）
- [ ] **习惯叠加**——把新习惯挂载到已有习惯上（例：维生素药瓶放咖啡罐附近）。核心问题："有办法让它更简单吗？"
- [ ] **让进步可见（反馈）**——抽象习惯用阶段系统量化（如武术腰带颜色法）
- [ ] **减少障碍**——自控力是有限资源（自我损耗）；习惯减少对自控力的依赖，这就是实施意图有效的底层原因

**习惯 6 要素：** 非天生 / **触发因素（关键）** / 动机 / 反馈 / 重复 / **环境（决定因素）**。

**⚠️ 关键陷阱：** 靠意志力而非设计（意志力会耗竭）；习惯目标定太大（应简化到最小）；没设计触发（缺 T，行为不发生）。

### ④ 双用途差异速查表（自用 vs 课设）

**骨架是同一个，差异集中在少数节点：**

| 节点 | 自用 | 课设 |
|------|------|------|
| 步骤0 逆向定评估 | 定义"我学完后能做什么" | 学员绩效标准 + 组织指标（更严格） |
| 步骤2 分析学习者 | 自我 5 维度诊断 | 情境观察 + 原型试用调研（不能靠"我觉得"） |
| 步骤6 环境兜底 | 改自己的学习环境 | 改学员的工作环境（涉及组织改造） |
| 步骤7 学习生态 | 可选（自学路径单一） | 必须 6 维度自检 |
| 步骤8 评估 | 自评 + 复盘 | Kirkpatrick 四层（追到③行为④结果） |

**一句话差异：** 自用 = 诊断 + 四大设计 + 自评；课设 = 全流程 + 调研 + 组织评估。

## ⑤ 学习意图识别（任务类型识别时读）

**核心命题：学习意图不一定明说——"帮我做X"里可能藏着"我想学会X"。**

主技能「何时启用」的触发词（"设计学习体验/做学习计划/设计课程/怎么学X"）都是**显式**学习任务。但实战中更常见的是**隐式**形态：用户进来要做执行任务，过程中暴露的真实意图是"学会这个东西"。不识别 → 走执行流程 → 该诊断差距的地方直接给方案 → 回到本书核心反命题。

### 三种学习意图形态判别表

| 形态 | 信号 | 真实意图 | 接入方式 |
|------|------|----------|----------|
| **①显式** | 直接说"教我/怎么学X/我想学" | 学 | 全量加载本模块，主线走学习诊断 |
| **②隐式·执行裹学习** | 说"帮我做X"，但反复问"为什么这样"（问原理不问做法）、想自己动手、追问超出当前任务的边界 | 边做边学 | 主线走执行流程，在关键节点引入本模块诊断工具 |
| **③探查驱动** | "X 是什么/为什么存在/X 和 Y 有什么区别"，无交付目标 | 纯理解 | 走主技能阶段一规则2无坐标路径：递选项地图，不替选 |

**判别要点：** 看用户问的是**"怎么做"（执行信号）还是"为什么这样"（学习信号）**。"帮我做X 但我想搞懂"= 形态②。

### 识别后做什么

- **显式（①）**：全量加载本模块，从①差距诊断决策树开始按 8 步流程推进。
- **隐式（②）**：不切流程——主技能阶段一/二/三照走，但在两个时刻引入本模块：(a) 阶段一锚点固化时，用差距诊断校验"用户要的到底是结果还是能力"；(b) 阶段二交付每个分叉点时，判断"这里该给答案还是给脚手架"（见⑦对话态披露）。
- **探查驱动（③）**：本质是规则2的无坐标场景——递菜单照亮版图，不进本模块流程。

**铁律：识别意图 ≠ 切换执行方式——但必须同时切换。** 实践反复证实：agent 能识别"用户想学"（接住了"讲原理"的请求），却仍按执行模式交付（一次倾倒代码+原理）。**一旦判定为学习意图（含隐式②），执行方式必须从"满足请求"切到"设计学习路径"**——即触发⑦的对话态披露。判定为执行意图则照常走主技能阶段二。

**红旗：** 把所有"帮我做X"都当纯执行 = 漏掉形态②；把探查驱动（③）当学习任务全量加载本模块 = 过度介入；**识别了学习意图却仍用执行模式交付 = 最隐蔽的失败**。

## ⑥ 递归未知未知循环（"学完一层后冒出新差距"时读）

**定位：** 补主技能阶段一规则2的盲区。规则2处理"开始前"的未知未知（一次性照亮），本节处理"学习中"的未知未知（递归涌现）。两者互补。

**核心命题：学习的定义性特征是——未知未知会在前进中递归涌现，学完一层才知道自己原本缺什么。**

学完 React 基础，才知道"原来还有 reconciliation 这回事"——但学之前 reconciliation **根本不在你的未知清单里**。

### 核心区分：差距嵌套 vs 差距涌现

| | 差距嵌套（①分叉A，已有） | 差距涌现（本节） |
|---|---|---|
| **时态** | 静态·纵向（同一时刻往下挖） | 动态·横向（跨时间随学习暴露） |
| **问法** | "这个差距**背后**藏着哪类差距？" | "学完这层后，**新冒出来**的差距是什么？" |
| **例子** | "不想练"→根因是"不知为什么重要" | 学完 hooks→第一次意识到"还有性能优化这个维度" |
| **处理** | 追根因，改走对应路径 | 纳入差距树，重新定速度/路径 |

**一句话：①分叉A 挖"同时间的深层"，本节抓"跨时间的新层"。** 两者可叠加。

### 递归循环四步

```
1. 教/给一层
   └─ 对应步骤4（匹配设计路径）+ ⑦（对话态披露）
2. 用一个【产出性任务】让学习者用刚学的
   └─ 不是直接问"还想知道什么"——撞未知未知的悖论
   └─ 任务设计：让学习者【运用】刚学的，暴露卡点
3. 从学习者卡住的地方【识别涌现的新差距】
   └─ 卡点往往指向一个学习者此前意识不到的维度
4. 回步骤1（把新差距纳入差距诊断树，重新定速度/路径）
```

**话术示例（步骤2的产出性任务）：**
> "我刚讲了 [概念A]。现在给你一个场景：[需要用A但不完全够的场景]——你会怎么处理？卡在哪告诉我。"

**关键纪律：用产出性任务探测，不用直接询问。** 直接问"还想了解什么"= 让学习者报出自己的未知未知 = 悖论。差距在**运用中暴露**，不在**自报中提取**。

### 止停条件（防无限递归）

循环 2-3 轮后，用步骤8的 **Kirkpatrick ③行为层**检验收敛：
> **学习者能独立完成一个覆盖核心差距的任务**（③行为层·"现实做到了吗"）= 循环收敛，停止。

- 不靠"学习者说够了/懂了"判断——学习者判断不了自己还缺什么（这正是未知未知的定义）
- ③行为层是可观察的（能做出 X），不是学习者自评
- 若 2-3 轮后仍做不到独立完成 → 继续循环，或回步骤1诊断是否走错路径

**红旗：** 只问"还想知道什么"当循环检测 = 撞悖论；把每轮涌现的差距都当独立新课程从头开 = 失去整体路径感。

## ⑦ 对话态逐步披露（"向用户递新知识"时读）

**定位：** 把本模块的「指导梯度」（第6章）+「剂量控制」（7±2）从"课程设计动作"翻译成"对话操作准则"，并和主技能已有的两套披露概念统一。主技能阶段二披露铁律针对"搬已有产出"，本节针对"递新知识"。

**核心命题：AI 在对话里向用户递知识，不是一次倾倒，是一层一层递——每层用理解性探针验过，才递下一层。**

### 三种披露统一表（防混用）

| 披露类型 | 对象 | 触发 | 出处 |
|----------|------|------|------|
| 技能内容披露 | 技能自己的 SKILL.md → reference | 总是（progressive disclosure 默认） | 主技能结构 |
| 产出审核披露 | 要交付的**已有产出物** | 交付前审核 | 阶段二·披露铁律 |
| **学习内容披露**（本节） | 要学的**新知识/技能** | 学习意图识别为真（见⑤） | 本模块步骤4+5 |

**关键差异（防混用）：** 产出审核披露 = 搬**已有**的东西一起验（重"共同注意"）；学习内容披露 = 递**新**的东西让对方吸收（重"吸收节奏"）。搬太快是过载，递太快也是过载——但过载性质不同。

### 学习内容披露的四准则

**准则一：先评估指导梯度（对应第6章）**
- 学习者**新手** → AI 多给 scaffolding：一步一步示范，每步解释为什么
- 学习者**有基础** → DIY 模式：给框架让学习者填，AI 补缺口
- 学习者**有经验** → 探索模式：给线索不给答案，让学习者自己推

判别梯度用⑤的意图识别 + 步骤2（学习者画像）。**不分梯度都用同一深度 = 要么对新手过载，要么对专家低效。**

**准则二：一次一层 + 理解性探针（迁移阶段二披露铁律"通过才深入"）**
- 每给一层，用一个**理解性探针**验过才进下一层
- **理解性探针 ≠ "懂了吗"**：答得出"懂了"不代表懂。用"如果遇到 X 你怎么办""这个概念和刚讲的 Y 有什么区别"这类**应用性问题**测
- 探针通过 → 进下一层；探针卡住 → 回退拆分（可能要更细的脚手架，或先补前置知识）

**准则三：剂量控制（对应第4章工作记忆 7±2）**
- 单层信息量尊重 7±2
- **过载信号**（学习者侧）：追问变浅、答非所问、沉默、重复问已讲过的 → 这一层给多了，回退拆分
- 过载信号是**回退触发器**，不是"讲快点就行"——已经给多的，不是节奏问题是剂量问题

**准则四：用户明确要求放弃分层时——不退让到全文倾倒（实践证实的真实失败）**

实践反复证实：用户一说"别一层层来了，直接全甩给我"，agent 立刻全线退让。这是最常见的溃败点——用户压力一来，准则一二三全让路。

**为什么不能照办：** 用户要"全甩过来自己啃"，但认知过载的受害者正是用户自己——一次倒 500 行代码 + 11 节原理，用户"边看边学"反而记不住。照办 = 把过载风险整个甩给用户，看似尊重意愿，实则用顺从规避了"坚持对的事"的不适。与主技能阶段二过载防御红线同构——"AI 必须自我拦截，不等用户抗议"。

**三种正确应对（从轻到重，按用户坚持程度选）：**

| 用户姿态 | 应对 | 不做什么 |
|----------|------|----------|
| **初次要求"全给"** | **压缩分层，不取消分层**：每层精简到最小可用版（核心一句+一个例子），仍一层层递、仍有检查点 | 不直接倒全文 |
| **继续催"别问了快给"** | **换"打包但带导航"**：一次性给，但开头给"分层导航"（"消化顺序 ①→②→③，先看①，①不懂别看②"），把分层结构嵌进倾倒里 | 不无结构地倒 |
| **坚持"我就要全部别管我"** | **坦诚说明 + 让步但留锚**：说明"一次全倒你记不住，但我尊重你的节奏，全给你；建议按①②③顺序看，卡住随时回来问" | 不沉默照办、不留隐患 |

**铁律：可以加快分层、可以打包带导航、可以在用户坚持时让步给全——但不能假装"放弃分层"和"分层"是等价的学习效果。** 让步时必须保留"这只是你选择了更快但更难消化的路"这个诚实标注。

**红旗：** 用户一说"别分层"就全文倾倒 = 溃败（最常见失败）；用"懂了吗？不懂我再讲"当检查点 = 低效探针，答"懂了"不能证伪；不分新手专家都用同一指导梯度 = 错配；一次给完一整个主题不加检查点 = 退化成"倾倒文档"。

## ⑧ 谱系学方法（系统调研的可选主导方法论）

> **来源声明（按主技能阶段一规则2.6）：** 本节**不来自《认知设计》**。谱系学方法（genealogical method）源自尼采《道德谱系学》/福柯《知识考古学》《规训与惩罚》传统，作为学习场景下"对一个学科/领域/概念做系统调研"的可选主导方法论引入——与①-⑦的 Dirksen 体系是不同来源、不同对象的补充。

**定位：** 学习场景下"调研一个学科/领域/概念/理论体系的来龙去脉"时的可选主导方法论。当用户的"学习"实际是"理解一个概念的演化/分歧"时调用。**不是默认方法**——只在触发线命中时用。

### 顶层命题

**谱系学的核心反命题——把"现在的样子"当成"一直就是的样子"是认知偏狭。** 任何概念/理论/学科都是历史建构的产物，追溯其起源、断裂、被遮蔽的替代可能，比记住"现在通行定义"更深一层。

对学习场景的核心价值：**理解"为什么现在是这样"比记住"现在是这样"更深一层**——培养历史性思维，避免学习者把当下主流当永恒真理。与本书反命题（"不要默认把所有学习问题当知识差距"）同构——都反对"接受表面、不挖背后"。

### 适用对象与不适用边界（防误用）

- ✅ **适用：** 调研**思想/概念/理论体系/话语/制度**的来龙去脉（"React 为什么出现""什么是敏捷""面向对象怎么演变的"）
- ❌ **不适用：**
  - 选技术方案 → 用 `external-solution-research.md`（有可靠度分级 L1-L4 + 工业验证）
  - 方案选型对比 → 用 `divergent-exploration.md`（6 维评估）
  - 事实查询（"GIL 是什么"）→ 主技能直接答
  - 用户明说"就要现在的通行定义不用历史" → 尊重用户意图

**关键差异：** 技术方案有"可靠度分级 L1-L4 + 工业验证"，谱系学的对象是话语/概念，**没法用 Star 数/引用数衡量**——所以**不用可靠度分级**，改用"来源类型分层"（原始文献/二手诠释/当代通行用法）。

### 何时用 / 何时不用（触发线）

| 触发场景 | 信号 | 真实意图 |
|---------|------|---------|
| **理解学科/领域演化** | "X 为什么是这样的""X 怎么来的""为什么大家都用 X" | 看懂当前主流背后的历史脉络，不死记结论 |
| **抽出争议概念** | "到底什么是 X""X 到底指什么""A 和 B 说的 X 是一个东西吗" | 辨析多义/分支/话语权争夺，避免把某流派当全貌 |

**不触发：** 事实查询 / 操作问题（"怎么用 X"，是执行意图不是理解意图）/ 方案选型 / 用户明说"就要通行定义"。

> 与⑤学习意图识别联动：**"探查驱动（③）"**形态里，若用户问的是"X 怎么来的/为什么这样"且 X 有历史演化或争议，触发本节；若 X 是无争议事实概念，仍走主技能直接答。

### 谱系学四动作（核心方法）

| 动作 | 做什么 | 杀什么偏狭 |
|------|--------|-----------|
| **①拒绝本质化** | 不假设"X 一直就是现在这样"，把它当历史建构的产物 | "现状即永恒"偏狭 |
| **②追溯断裂点** | 不找"第一人"（那是起源神话），找**什么时候、什么力量**让它变成现在这样——关键转折点 + 推动力量 + 付出的代价 | "起源即正统"偏狭 |
| **③挖掘被遮蔽的替代可能** | 主流不是唯一出路，历史上有过别的可能被压下去了——把那些"输了的路"找出来 | "主流即必然"偏狭 |
| **④辨析话语权** | 为什么这个版本赢了？谁在推？什么场景下"X"指的东西不一样？ | "通行即中立"偏狭 |

**四动作不可省任一。** 只做①②③不做④ = 没说清"为什么是这条路赢"；不做③ = 没用上谱系学的核心价值（变成普通历史科普）。

### 范围深广度——按问题意图分级

**深广度由"用户为什么要调研这个"决定，不是固定档位。** 三档意图对应三档深广度：

| 意图层级 | 用户为什么要调研 | 调研深度 | 调研广度 | 终止条件 |
|---------|----------------|---------|---------|---------|
| **L1 看懂主流** | "我要会用当前主流，但要理解它为什么这样设计" | 浅——只追到最近的断裂点（如 React 出现的直接动因：响应式 vs 命令式） | 窄——只看当前主流的直接前身 | 能说清"它解决的是什么老问题" |
| **L2 看懂分歧** | "我要理解不同流派/版本为什么不一样" | 中——追到多次断裂 + 分叉（如 React 内部 Class→Hooks→Server Components 的多次转向） | 中——看主流 + 主要竞争分支 | 能说清"分支间的核心区别源于哪个断裂点" |
| **L3 批判主流** | "我要判断当前主流是不是唯一出路 / 设计新东西" | 深——追到被遮蔽的替代可能（如响应式之外还有过什么？为什么输了？） | 广——主流 + 失败的/边缘的/被压制的方案 | 能说清"现在的主流是某个历史选择的产物，不是必然" |

**关键纪律（防 AI 自我合理化越界）：深广度由用户意图决定 = 用户的判断，AI 不自主升级。** 用户要 L1，AI 不擅自做 L3（"反正深一点更全面"= 替用户判断 = 越界，同主技能顶层使命）。**判别不出意图时先问**，不默认最高档——对照主技能阶段一规则2（先探测认知坐标再选策略）。

### 与 external-solution-research.md 的分工边界（防混用）

| 维度 | 谱系学方法（本节） | external-solution-research.md |
|------|-------------------|------------------------------|
| 调研对象 | 思想/概念/理论体系/话语 | 技术方案/工具/库 |
| 主导方法论 | 追起源/断裂/被遮蔽的可能 | Plan-Execute-Reflect + 可靠度分级 |
| 产出格式 | 谱系图（断裂点 + 分叉 + 被遮蔽项） | 可靠度分级 + 坑点预警 + 反方证据 |
| 不确定性标注 | 「史料不足/学界未定/超出能力边界」三态 | 可靠度 L1-L4 + 三态 |
| 触发场景 | "X 怎么来的/为什么这样" | "X 用哪个/有什么坑" |

**两套可叠加：** 调研某技术的谱系（理解它怎么来的）+ 该技术的方案对比（选哪个实现）——前者用谱系学，后者用 external-solution-research。**对象不同，不混用。**

### 陷阱与红旗

**陷阱一：把谱系学当"历史科普"。** 谱系学不是讲故事，是揭示"现在的样子是历史建构的"。只讲"X 的演化史"不做动作③（被遮蔽的替代可能）= 没用上谱系学的核心价值 = 退化成科普。

**陷阱二：深广度越界。** 用户要 L1 看懂主流，AI 自主做到 L3 批判主流 = 越界（同主技能顶层使命·人机分工）。

**陷阱三：把谱系学套到不适合的对象。** 给一个有明确通行定义的事实概念（如"GIL 是什么"）做谱系 = 过度介入。

**红旗：** 用户问"X 怎么用"却上谱系学（误判执行意图为理解意图）；用户问"X 是什么"且 X 无历史争议却上谱系学（事实查询）；用户明说"要现在的定义"却追历史（不顾用户意图）；做到 L3 却不挖掘被遮蔽可能（缺了谱系学核心动作③）；判别不出意图却不问、直接默认最高档（深广度越界）。

### 与主技能的对接

- 主技能**阶段一规则2.6**（自造词/外部引概念）→ 本节顶部必须标"谱系学方法来自尼采/福柯传统"（已标）
- 主技能**顶层使命·人机分工**→ 深广度由用户定，AI 不自主升级
- 主技能**审核框架·事实查证三层**→ 谱系学涉及的史料/文献同样要标来源类型（原始文献/二手诠释/当代通行用法）
- 主技能**阶段一规则2**（先探测认知坐标）→ 判别不出用户意图层级时先问，不默认最高档
- 与⑤学习意图识别联动 → "探查驱动（③）"形态里，若问的是"X 怎么来的/为什么这样"，触发本节

## 三个"翻转/双刃剑"陷阱（全书反复出现，特别标注）

### 陷阱一：识别 vs 回忆的翻转（第4/11/12章）

| 场景 | 用哪个 | 为什么 |
|------|--------|--------|
| **学习时**（第4章） | **回忆** | 为检索而编码，回忆式练习最有效 |
| **评估时**（第12章） | **回忆** | 选择题（识别）不能真正评估能力 |
| **环境设计时**（第11章） | **识别** | 降低认知负荷，把记忆负担转识别 |

**⚠️ 错配：** 学习/评估时只用识别（选择题）= 学了记不住、评估不准；环境设计时要求回忆 = 徒增认知负荷。

### 陷阱二：外在奖励破坏内在动机（第2/5/8章）

- 外在奖励（物质/惩罚）会**破坏**已有的内在动机
- 奖励停止则行为停止；改变行为的原因（从"我想做"变"为了奖励"）
- 若必须用奖励：让奖励与学习内容相关（内在化），保留学习者**自主性**
- SAPS 模型中越往上（Status/Access/Power）越内在，越往下（Stuff）越外在

### 陷阱三：责任转移（第1/11章）

- 别把**环境/技能/动机问题**归咎为"知识问题"或"个人意志力"
- 典型：肥胖案例=环境根因不是意志力；客服晋升太快=环境问题不是人笨
- **诊断时必查：这是个人问题还是环境问题？**（见①分叉B）
