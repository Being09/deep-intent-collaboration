# 深度意图协作（Deep Intent Collaboration）

> **你在用 AI 写代码、做设计、做技术选型时，是否遇到过这些情况？**
>
> - 🤖 AI 没问清楚需求就直接开干，做出来不是你要的
> - 🤖 AI 替你做决策——"我帮你选了 X"——而你其实没想好
> - 🤖 AI 一次倒出一大堆内容，信息过载，看不完
> - 🤖 AI 做完就走，没有复盘，下次遇到同样问题又重来

这是一个**适用于任意支持 Skills 规范的 AI Agent**（Claude Code、Codex、Cursor、OpenCode 等）的 Skill，它把「人机协作中的意图对齐」提炼为一套可操作的三阶段流程，用来解决上面这些问题。

---

## 核心命题：增强人类，而非增强 AI

AI 在执行类协作中常见的失败模式——AI 自作主张跳过澄清、替用户在分叉点做决策、一次倒出过载的结果、在结束时跳过复盘——**本质都是「替代人」而非「增强人」**。

本 Skill 的终极目标不是让 AI 产出更快更多，而是让**人**在协作结束后变得更清醒：

> 知道自己**要什么**、**为什么这么选**、**关心什么**、**下次怎么做得更好**。

Skill 不替你决策，而是引导协作走完三个阶段：

| 阶段 | 核心命题 | 关键动作 |
|------|---------|---------|
| 一·意图澄清 | 意图是**构造**出来的，不是提取出来的 | Purpose → Constraints → Success criteria → 本质需求锚点 → 逻辑自洽校验（五层递进） |
| 1.5·发散探索 | 没看清全景就选，等于盲选 | 双向搜索（内/外）→ 6 维评估 → 差异/影响分析 → 逻辑校验收官 |
| 二·执行对齐 | 对齐不是一次性的，是持续的 | 快速对齐（常规推进，轻量反查）vs 分叉点校验（四信号触发，停下让人选） |
| 三·复盘沉淀 | 复盘检验的是「人有没有变清醒」 | 偏移回溯 → 选择回溯 → 考量系统化 → 情绪/状态回溯 → 经验提炼 |

## 核心用户价值

| 你的痛点 | 技能怎么解决 | 你得到什么 |
|----------|-------------|-----------|
| AI 没问清楚就开干，方向错了 | 阶段一「意图澄清」五层递进，从 Purpose 追到本质需求 | **第一次就做对方向** |
| AI 替你做决策（"我推荐 X"） | 阶段二「分叉点校验」碰到选择就停下，让你选 | **决策权在你手里** |
| AI 一次倒出太多，看不完 | 「过载防御」按约定的深度边界交付，不超纲 | **逐步消化，不淹死** |
| 不确定选什么技术/方案 | 阶段 1.5「发散探索」双向搜索 + 6 维对比 + 差异分析 | **看清选项差异再做选择** |
| AI 分析是黑箱，不知道依据 | 「决策透明化框架」标注推理来源、确信度、信息溯源 | **每个结论都有据可查** |
| 协作完 AI 就走，下次又重来一次 | 阶段三「复盘沉淀」+ 跨会话经验记忆层 | **经验可积累，越用越聪明** |
| AI 方案对比只抛矩阵不说区别 | 系统化评估 + 差异分析 + 影响分析 | **不是只看表，是看懂区别** |
| 选完方案就动手，没经逻辑校验 | 「审核框架」三段论 + 反证法检验选择逻辑 | **选择有依据，不是拍脑袋** |

> 一句话说清：**这是一个让 AI 从"替你做"变成"帮你想清楚再做"的协作规范。** 它不加速 AI 产出——它**减少返工**。方向做错、信息过载、替决策导致的返工，比多花五分钟澄清需求慢得多。

## 四类危险信号（见到立即拦截）

这套 Skill 会在你发出以下信号时**主动拦住 AI**，把你拉回对话框：

| 用户信号 | 危险类型 | 立即应对 |
|---|---|---|
| "直接给我"、"别问那么多" | 跳过意图澄清 | 反问 Purpose，不给代码/方案 |
| "X vs Y 哪个好" | 接受解法框架 | 回到需求层：什么项目、什么场景 |
| "你随便选"、"我相信你" | 分叉点替决策 | 停下标注分叉，让你选并追问理由 |
| "行就这样吧"、"功能做完了" | 复盘逃避 | 主动发起复盘 |

## 谁该用 / 不该用

| 你如果…… | 建议 |
|----------|------|
| 经常让 AI 做功能、设计方案、技术选型 | **每次都用** — 防止 AI 跑偏 |
| 担心自己一开始没想清楚约束，怕 AI 过度产出 | **必用** — 帮你把模糊意图逼出来 |
| 希望每次协作后能留下可复用的经验 | **必用** — 跨会话记忆闭环 |
| 只查事实、改错字、做纯执行活 | **不用** — 直接对话更快 |

**不适用场景**（直接对话即可，无需启动本 Skill）：

- 事实查询（"Python GIL 是什么"）
- 低风险可逆操作（改名、加注释、修错字）
- 你已经想得非常清楚、只差执行的纯体力活

## 使用方式

安装后通过**主动调用**激活——在 Agent 中输入：

```
/deep-intent-collaboration    # 英文名
/深度意图协作                   # 中文名
```

### 一次典型协作的节奏

```
你: /深度意图协作 帮我写一个用户通知功能

AI: 先确认你的本质需求 → 划边界 → 定成功标准 → 锚点固化
  ↳ 你回答几个问题（每层一个问题，不淹没你）

AI: 需要探索通知方案 → 双向搜索（内/外）→ 6 维对比 + 差异分析
  ↳ 你看到对比矩阵和方案差异，自己做选择

AI: 开始写代码 → 每步交付一小块等你确认
  ↳ 你边看边调整，不会一次性被倾倒

AI: 功能做完 → 发起复盘 → 问你是否记下经验
  ↳ 你带走可复用的认知，下次用得上
```

## 技能结构一览

```
┌─────────────────────────────────────────────────────────────────────┐
│                    决策透明化框架（横切贯穿）                          │
│  🤖推理依据 · 🟢确信度 · 🔗信息溯源 · 👤人机分工                     │
├─────────────────────────────────────────────────────────────────────┤
│  阶段一         阶段 1.5             阶段二           阶段三          │
│  意图澄清  →    发散探索增强      →  执行对齐     →  复盘沉淀        │
│                 ·双向搜索(内+外)      ·快速对齐       ·五动作         │
│                 ·6维评估+差异分析     ·分叉点校验     ·落盘三步       │
│                 ·逻辑校验收官        ·产出审核(自顶向下)              │
├─────────────────────────────────────────────────────────────────────┤
│                    审核框架（横切贯穿）                               │
│  方案审核（选前）| 产出审核（交付前）| 自洽审核（全程）               │
└─────────────────────────────────────────────────────────────────────┘
```

两个横切机制（审核框架、决策透明化框架）贯穿所有阶段，是本次升级新增的「深度」所在：前者保证选择经得起逻辑推敲，后者保证每个结论都可追溯。

详见 [`SKILL.md`](./SKILL.md) —— 1100+ 行的完整协作规范。
