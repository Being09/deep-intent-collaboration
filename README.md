# 深度意图协作（Deep Intent Collaboration）

**你在用 AI 写代码、做设计、做技术选型时，是否遇到过这些情况？**

- 🤖 AI 没问清楚需求就直接开干，做出来不是你要的
- 🤖 AI 替你做决策 — "我帮你选了 X" — 而你其实没想好
- 🤖 AI 一次倒出一大堆内容，信息过载，看不完
- 🤖 AI 做完就走，没有复盘，下次遇到同样问题又重来

**这个 Skill 就是解决这些问题的。**

它不教 AI "更聪明地干活"——而是让 AI 在协作中**停下来、让你想清楚、交给你决策、最后帮你沉淀经验**。

---

## 谁该用这个 Skill

| 你如果…… | 建议 |
|----------|------|
| 经常让 AI 做功能、设计方案、技术选型 | **每次都用** — 防止 AI 跑偏 |
| 担心自己一开始没想清楚约束，怕 AI 过度产出 | **必用** — 帮你把模糊意图逼出来 |
| 希望每次协作后能留下可复用的经验 | **必用** — 跨会话记忆闭环 |
| 只查事实、改错字、做纯执行活 | **不用** — 直接对话更快 |

## 核心用户价值

| 你的痛点 | 技能怎么解决 | 你得到什么 |
|----------|-------------|-----------|
| AI 没问清楚就开干，方向错了 | 阶段一「意图澄清」五层递进，从 Purpose 追到本质需求 | **第一次就做对方向** |
| AI 替你做决策（"我推荐 X"） | 阶段二「分叉点校验」碰到选择就停下，让你选 | **决策权在你手里** |
| AI 一次倒出太多，看不完 | 「过载防御」按照约定的深度边界交付，不超纲 | **逐步消化，不淹死** |
| 不确定选什么技术/方案 | 阶段 1.5「发散探索」双向搜索 + 6 维对比帮你看到全景 | **看清选项差异再做选择** |
| AI 分析是黑箱，不知道依据 | 「决策透明化框架」标注推理来源、确信度、信息溯源 | **每个结论都有据可查** |
| 协作完 AI 就走，下次又来一次 | 阶段三「复盘沉淀」+ 跨会话经验记忆层 | **经验可积累，越用越聪明** |
| AI 方案对比只抛矩阵不说区别 | 系统化评估 + 差异分析 + 影响分析 | **不是只看表，是看懂区别** |
| 选完方案就上线，没经过逻辑校验 | 「审核框架」三段论 + 反证法检验选择逻辑 | **选择有依据，不是拍脑袋** |

---

## 一句话说清

> **这是一个让 AI 从"替你做"变成"帮你想清楚再做"的协作规范。**

它不加速 AI 产出——它**减少返工**。慢的是浅协作——方向做错、信息过载、替决策导致的返工，比多花五分钟澄清需求慢得多。

## 快速开始

```bash
# 安装
npx skills add Being09/deep-intent-collaboration

# 在对话中使用
/deep-intent-collaboration    # 英文名
/深度意图协作                   # 中文名
```

### 一次典型协作的节奏

```
你: /深度意图协作 帮我写一个用户通知功能

AI: 先确认你的本质需求→划边界→定成功标准→锚点固化
  ↳ 你回答几个问题（每层一个问题，不淹没你）

AI: 需要探索一下通知方案 → 双向搜索（内/外）→ 6 维对比
  ↳ 你看到对比矩阵+差异分析，做选择

AI: 开始写代码 → 每步交付一小块等你确认
  ↳ 你边看边调整，不会一次性被倾倒

AI: 功能做完 → 发起复盘 → 问你是否记下经验
  ↳ 你带走可复用的认知，下次用得上
```

### 不适用场景

- Python GIL 是什么？→ 直接问
- 把这个变量名改一下 → 直接改
- 需求已经想死了，只差敲代码 → 直接做
